## 1. seed 注入确认（逻辑严密档前置）

### 1.1 现状（已 grounding）

- 随机种子**硬编码**在 [sensor_model.py](file:///Users/bytedance/coding/AUV-Master-Mag/src/auv_mag_tracking/sensor_model.py) 各传感器构造函数默认值：`SensorModel(random_seed=7)`、子类 `31`、`13`、`NavigationSimulator(random_seed=23)`、`19`、`SonarModel(random_seed=23)`。
- [recorder.py](file:///Users/bytedance/coding/AUV-Master-Mag/src/auv_mag_tracking/viz/recorder.py) 的 `simulate_run(scenario, deployment_mode, max_steps, duration_override_s)` **签名不含 seed**。
- [main_viz.py#L534-L569](file:///Users/bytedance/coding/AUV-Master-Mag/src/auv_mag_tracking/main_viz.py#L534-L569) 的 `AuvCableTrackingSimulation.__init__` 内逐个实例化传感器：`MagnetometerModel(scenario.sensor)`、`SonarModel(scenario.sonar)`、`IMUSimulator(scenario.sensor)`、`NavigationSimulator(scenario.navigation)`——**均未传 seed**，因此每次运行用相同硬编码种子，结果确定性可复现但**无法多种子**。

### 1.2 结论

`dr_ins_boundary_sweep.py` 当前**不支持显式 seed**；同一场景多次运行结果完全相同。要做"3 seed × 场景"统计，必须让种子可注入。

### 1.3 所需最小 src 改动（执行逻辑严密档时单独提出确认）

最小侵入方案（三处转发，不改算法逻辑）：

1. `AuvCableTrackingSimulation.__init__` 增加可选 `seed_offset: Optional[int] = None`，若提供则在实例化各传感器时把 `random_seed = base_default + seed_offset` 传入（各传感器构造函数本就接受 `random_seed`）。
2. `simulate_run(...)` 增加可选 `seed_offset` 形参，透传给 `AuvCableTrackingSimulation`。
3. `dr_ins_boundary_sweep.py` 增加 `--seeds N`（或 `--seed-offsets a b c`），对每个 seed 跑一遍并把 seed 列写入 CSV。

> **执行约定**：该改动在跑逻辑严密档时作为独立编辑提出；若用户不希望改 src，则逻辑严密档退化为"多场景 n=1 + A/B 对照"，并在 docs/27 明确标注 n=1。

## 3. 指标定义（FIELDNAMES 13 列）

来自 [dr_ins_boundary_sweep.py](file:///Users/bytedance/coding/AUV-Master-Mag/tools/dr_ins_boundary_sweep.py) 的 `_metrics_row`：

| 列 | 含义 | 来源 |
|---|---|---|
| `case` | 场景名 | scenario.name |
| `sonar_prob` | 声呐检测概率 override（空=用场景默认） | CLI/网格 |
| `health` | 综合健康得分 | `health_score(metrics)` |
| `route` | 路径完成比 | `route_completion_ratio` |
| `endpoint` | 是否到达终点（≥0.5 计 1） | `endpoint_completed` |
| `geometry` | 迷宫几何是否通过（≥0.5 计 1） | `maze_geometry_passed` |
| `max_jump_m` | route progress 最大单步跳变（m） | `route_progress_max_jump_m` |
| `large_jumps` | 大跳变次数 | `route_progress_large_jump_count` |
| `track_xt_m` | TRACK 段平均横偏（m） | `track_mean_cross_track_m` |
| `vehicle_err_deg` | 平均车辆航向误差（°） | `track_mean_vehicle_heading_error_deg` |
| `final_xt_m` | 终点横偏（m） | `final_cross_track_m` |
| `mag_path_fraction` | 磁路径观测占比 | `magnetic_path_observation_fraction` |
| `passed` | 综合判定 | 见下 |

**passed 判据**：`endpoint_completed >= 0.5 AND maze_geometry_passed >= 0.5 AND route_progress_large_jump_count == 0`。
