> **结论（诚实记录，与原占位预期相反）**：在 HEAD 基线下，单独切换 progress guard **几乎不改变**这四个边界场景的结果——sparse/dropout 两态完全一致；continuous-sonar 仅有 77.6↔77.9m 的噪声级差异，`large_jumps` 与 pass/fail **均无翻转**。
>
> **机理**：guard 通过 `advance_nominal_route_progress` 在每帧用导航位置 seed 进度窗口（[orchestrator.py L558-561](file:///Users/bytedance/coding/AUV-Master-Mag/src/auv_mag_tracking/perception/orchestrator.py#L558-L561)），再把投影限制在 `[progress-lookback, progress+lookahead]=[−8m, +30m]` 窗口内。当车辆**沿名义路线单调推进、且最近投影点本就落在该窗口**时，窗口投影与全局最近点投影**返回同一点**，guard 退化为恒等操作。本批失效不是"投影跳到远端 lane"（那才会被窗口拦截），而是**先验几何被平移/旋转/缩放扭曲后整体偏移**——属于修正层（route-prior-correction）的可观测性问题，guard 这一几何安全网管不到。
>
> **对论文的价值**：这是一条**反事实/消融阴性结论**——它把"失效根因"从"进度窗口几何约束"**排除**，定位到"先验修正在弱观测（continuous-sonar 无磁路径、sparse 锚点稀疏）下不收敛"。结合 §2.1 中 dropout（`mag_path_fraction=0.98`）唯一通过，证明**真正的因果改善变量是磁路径观测覆盖率，而非 progress guard**。建议论文据此把 guard 定位为"防远端误投影的安全网"，把 dropout 的磁路径在线率定位为"边界可通过性的主因"。

**失效边界二维结论**：

1. **sonar_dropout 全 tier 通过**（light/mid/heavy 均 `large_jumps=0`、`max_jump≤0.3m`、`mag_path_fraction=0.98`）——磁路径观测在线是边界可通过性的决定因素，与 §2 阴性消融结论互证。
2. **continuous sonar 边界在 light↔mid 之间**：light 通过（0.8m/0 跳），mid 即失效（77.6m/2 跳），heavy 维持失效（80.0m/2 跳）。即 continuous-sonar 无磁路径冗余时，先验偏差一旦进入 mid 档就触发跨 lane 跳变。
3. **sparse (tier×prob) 平面高度非单调**：失效区**不规则散布**而非随 prob 单调——sparse_light@0.10 通过但 @0.15/0.20/0.30 全失效；sparse_heavy@0.20 反而是该 tier 唯一通过点（0.4m/0 跳），而 @0.10/0.15/0.30 出现 755.8m / 290.9m / 324.6m 的极端跳变。这直接复现 [project_memory lesson](file:///Users/bytedance/.trae-cn/memory/projects/-Users-bytedance-coding-AUV-Master-Mag/project_memory.md) 记录的"非单调相关、收紧门限反致横偏严重恶化（500m+）"——sparse_heavy@0.10 的 483.4m final_xt 即此现象的实测证据。
4. **route 完成度与几何通过解耦**：除 sparse_light@0.20（route=0.619）与 sparse_heavy@0.10（route=0.617）两例提前终止外，其余 16 行 `route≈0.995`——失效几乎全是"走完了但几何跨 lane"，不是"没走完"。
