# 代码简化与创新点凸出路线图

## 1. 背景与目标

当前代码库已经覆盖声磁协同电缆跟踪的多个阶段：声呐锁定、磁异常检测、先验电缆地图修正、主动 zig-zag 探查、埋深估计、离线实验、可视化与部署接口。功能完整度提高的同时，也带来一个论文写作和工程维护上的问题：如果所有状态机、探索性脚本和历史接口都并列呈现，核心创新会被实现细节稀释。

本路线图的目标不是在本轮大规模删除代码，而是明确哪些模块构成创新主线、哪些模块服务实验复现、哪些模块应作为 legacy/experimental 保留，后续再逐步收敛。

## 2. 当前创新主线

### 2.1 在线先验修正

核心问题是：现实电缆地图存在平移、旋转、尺度和局部形变误差，单纯依赖全局最近点投影容易把 AUV 拉向错误电缆段。

保留重点：

- 扭曲先验构造与修正。
- `PriorAlignmentState` / `PriorAlignmentEstimator`：把声呐观测残差显式转成先验到真实电缆的平移/旋转配准状态，并为后续磁路径电缆点接入保留同一更新接口。
- `nominal_route_prior_observation_correction_enabled` 对任务健康度的影响。
- 30 m 曲率核验中证明 baseline 并非无扭曲先验。

论文表达中应把它写成“几何先验在线校准机制”，而不是若干配置开关的堆叠。D4 map-frame 只保证投影连续；`PriorAlignmentState` 才承担把错位先验逐步拉向真实电缆的物理对齐。

### 2.2 zig-zag 主动探查与埋深估计

zig-zag 的价值不应只写成“控制轨迹变成之字形”。更准确的表述是：AUV 主动制造横向可观测性，使磁场强度、横偏变化和相位/路径证据在时间上形成可估计结构。

保留重点：

- TRACK 阶段的 zig-zag 激励。
- cycle-level 观测窗口。
- 埋深反演结果与 DL/T 1278 的 0.15 m 目标线。
- “未达标”时的诚实结论：当前 cycle-local 后验仍有误差瓶颈，但幅值增强呈现改善趋势。

### 2.3 跨 lane 几何安全

跨 lane 匹配是最容易被攻击的实验点。当前路线应把 progress guard、磁路径观测和 map-frame shadow 统一到“几何安全约束”叙事中。

保留重点：

- `route_progress_max_jump_m`
- `route_progress_large_jump_count`
- `lane_shortcut_indicator`
- `CableMapFrameTracker`

map-frame 方案已完成第一轮闭环接入：`CableMapFrameTracker` 进入 `MagneticCablePerception.nominal_route_projection()`，控制器仍通过统一的 perception provider 获取路线投影，避免在 controller 内复制几何修正逻辑。70/50 m 代表点显示 baseline 不被破坏，闭环 map-frame 最大跳变保持约 0.2 m；但关闭在线先验修正时任务仍失败，说明它修复的是投影连续性，不替代先验修正承担物理对齐。

为避免这一边界只停留在文字层，本轮已将在线先验修正显式模块化为 `src/auv_mag_tracking/perception/prior_alignment.py`，并新增单元测试覆盖残差门控与单步限幅。lane 代表点中 baseline 的 prior alignment 接受率为 1.000，最终累计约 7.53 m / -3.18° 的配准修正；关闭在线修正时该状态不更新。

### 2.4 plug-and-play API 与部署化

部署化本身是工程创新的一部分：算法不再依赖 `main_viz.py`、仿真真值或 Mock 数据，而是通过稳定输入输出契约接入外部系统。

保留重点：

- `src/auv_mag_tracking/api/`
- `tools/validate_deployment_inputs.py`
- `tools/export_cable_ops_report.py`
- `docs/31_声磁协同算法部署输入输出规范.md`

## 3. 建议代码分层

### 3.1 核心算法层

建议保留并重点维护：

- `src/auv_mag_tracking/perception/`
- `src/auv_mag_tracking/controller.py`
- `src/auv_mag_tracking/math_utils.py`
- `src/auv_mag_tracking/environment.py`
- `src/auv_mag_tracking/config/`
- `src/auv_mag_tracking/api/`

这一层应尽量减少对可视化、论文实验脚本和具体仿真入口的依赖。

### 3.2 实验复现层

建议保留，但按实验主题命名清晰化：

- `tools/*_sweep.py`
- `tools/render_*_figures.py`
- `results/YYYYMMDD_*`

这些脚本服务论文表格、图和回归证据。它们可以依赖仿真与评估真值，但必须在命名和文档中明确自己不是在线部署入口。

### 3.3 文档图层

建议继续保持：

- `docs/figure/`
- `docs/28_声磁协同方法论合龙.md`
- `docs/29_声磁协同实验设计与结果.md`
- `docs/30_声磁协同意义与展望.md`
- `docs/31_声磁协同算法部署输入输出规范.md`
- `docs/32_代码简化与创新点凸出路线图.md`

论文引用图应统一存入 `docs/figure/`，文档中使用 `figure/` 相对路径，不再引用版本库外的 results 目录。

### 3.4 legacy / experimental 层

建议暂时保留：

- `src/auv_mag_tracking/experimental/`
- 旧连接器、外部仿真器 mock 和硬件适配器。

这些模块可能仍包含接入经验，不应在本轮为了“看起来简洁”而删除。后续应在 API 稳定后，把仍有价值的适配器改成新 API 的 adapter。

## 4. 本轮不删除清单

本轮不建议删除：

- 任何会影响全量回归的状态机。
- 当前 docs/28-32 引用的实验脚本。
- `experimental/` 中尚未迁移到 `api/` 的连接器。
- recorder 与 metrics 中用于论文复现实验的通道。

原因是当前任务重点是补强实验逻辑、API 和部署文档，而不是重构主循环。过早删除会增加回归风险，也会削弱论文复现链条。

## 5. 后续可归档候选

在完成全量回归、文档引用稳定和 API facade 持续使用后，可逐步归档：

- 被新 `tools/*_sweep.py` 替代的临时诊断脚本。
- 不再被 docs/28-32 引用的旧图生成脚本。
- 与 `CableMap`、`NavigationInput`、`MagneticInput`、`SonarInput` 重复的 experimental 数据结构。
- 只服务早期调试、没有单元测试且没有文档引用的状态输出通道。

归档方式建议优先移动到 `docs/archive` 或 `tools/archive`，并在一个独立 commit 中完成，避免与算法改动混在一起。

## 6. 与论文叙事的对应关系

| 论文叙事 | 代码锚点 | 实验证据 |
|---|---|---|
| 在线先验修正 | `config/`、`perception/`、`radius_boundary_sweep.py` | radius integrity、radius causality |
| 主动 zig-zag 探查 | tracking 配置、burial inversion、recorder 通道 | zig-zag burial sweep |
| 跨 lane 几何安全 | progress guard、magnetic path、`CableMapFrameTracker` | lane shortcut stress |
| 部署与运维价值 | `api/`、validate/export tools | docs/31、API contract tests |

后续写作应围绕这些主线组织，而不是逐个介绍状态机和脚本。状态机只在必要时作为实现细节出现。

## 7. 后续重构顺序建议

1. 冻结当前论文实验脚本和结果表，确保可复现。
2. 把 API facade 后方逐步接入更完整的 `MagneticCablePerception`。
3. 将 `experimental/` 中仍有价值的外部连接器改写为 API adapter。
4. 删除或归档无文档引用、无测试、无部署路径的临时脚本。
5. 最后再考虑状态机合并或删减，避免在论文实验尚未稳定时引入行为漂移。

## 8. 本轮已执行剪枝

- 删除慢速半成品工具 `tools/map_frame_shadow_tuning.py`：D4 闭环后，map-frame 指标已由 `RunRecord` 直接记录，继续保留该调参网格会增加维护成本。
- 精简 `tools/lane_shortcut_stress_sweep.py`：移除 `map_frame_constraint_shadow` 伪变体及 baseline 复制逻辑，只保留真实消融项。
- 新增 `prior_alignment.py`：这不是增加一条新机制，而是把原本散落在 orchestrator 中的在线先验修正整理成可门控、可记录、可单测的独立状态。
- 保留核心复现入口：`zigzag_burial_sweep.py`、`radius_causality_sweep.py`、`lane_shortcut_stress_sweep.py`、部署校验和运维导出 CLI 均继续保留。
- 验证结果：API 导入、部署 CLI help、关键测试与全量测试均通过；本轮新增测试后全量 `pytest` 为 154 passed。
