# 调优指南

## 1. 调优方法论

### 1.1 调优循环

```
观察 → 假设 → 修改 → 验证 → 迭代
```

**详细步骤**：
1. **观察**：运行仿真，记录性能指标
2. **诊断**：分析瓶颈，识别根本原因
3. **假设**：提出参数修改假设
4. **实施**：修改配置，最小化改动
5. **验证**：重新运行，对比效果
6. **迭代**：基于结果继续调整

### 1.2 调优层次

```
第一层：信号处理层
    ↓
第二层：峰值检测层
    ↓
第三层：拟合与估计层
    ↓
第四层：控制决策层
    ↓
第五层：系统级调优
```

**原则**：从底层开始，逐层向上调优。

## 2. 信号处理层调优

### 2.1 带通滤波器调优

**症状**：
- AC 信号检测失败
- SNR 过低
- 频率估计不准

**调优参数**：
```python
signal.bandpass_half_width_hz = 8.0  # 默认值
```

**调优指南**：
| 场景 | 推荐值 | 理由 |
|------|--------|------|
| 纯净环境 | 4-6 Hz | 更窄带宽，更好噪声抑制 |
| 标准环境 | 8-10 Hz | 默认值，平衡 |
| 干扰环境 | 12-15 Hz | 更宽容，容忍频率漂移 |

**验证方法**：
- 查看 FFT 频谱，目标频率应在通带中心
- SNR 应 > 6 dB
- 频率估计误差 < 2 Hz

### 2.2 RMS 窗口调优

**症状**：
- 包络波动过大
- 响应延迟过长

**调优参数**：
```python
signal_processing.rms_cycle_count = 1  # 默认值
```

**调优指南**：
| 周期数 | 优点 | 缺点 | 适用场景 |
|--------|------|------|----------|
| 1 | 快速响应 | 波动大 | 高动态环境 |
| 2 | 平衡 | 平衡 | 默认 |
| 3+ | 平滑 | 响应慢 | 稳定环境 |

**验证方法**：
- 检查包络曲线，应平滑但不延迟
- 峰值检测应有足够的超前量

### 2.3 包络平滑调优

**症状**：
- 跟踪强度抖动
- 过峰检测不稳定

**调优参数**：
```python
tracking.envelope_time_constant_s = 0.5  # 默认值
```

**调优指南**：
| 时间常数 | 效果 | 适用场景 |
|----------|------|----------|
| 0.2-0.3s | 快速响应、略抖 | 快速机动 |
| 0.5s | 平衡 | 默认 |
| 0.8-1.0s | 平滑、延迟 | 稳定跟踪 |

**验证方法**：
- Signal View 中查看包络曲线
- 应该平滑但不明显延迟于真实信号

## 3. 峰值检测层调优

### 3.1 灵敏度调优

**症状**：
- **漏检**：真实峰值未触发
- **误检**：噪声触发假峰值

**调优参数**：
```python
tracking.min_peak_strength_nt = 18.0  # 最小峰值强度
tracking.turn_trigger_ratio = 0.85    # 触发比例
```

**调优策略**：

**漏检问题**：
- 降低 `min_peak_strength_nt`（但 > 3× 噪声）
- 降低 `turn_trigger_ratio`（如 0.80）
- 减少 `peak_ascending_min_samples`（如 1）

**误检问题**：
- 提高 `min_peak_strength_nt`
- 提高 `turn_trigger_ratio`（如 0.90）
- 增加 `peak_ascending_min_samples`（如 3）
- 增加 `hysteresis_fraction`（如 0.12）

### 3.2 冷却期调优

**症状**：
- 一次穿缆触发多次峰值
- 转弯响应慢

**调优参数**：
```python
tracking.peak_cooldown_s = 0.8  # 默认值
```

**调优指南**：
| 冷却期 | 效果 | 适用场景 |
|--------|------|----------|
| 0.3-0.5s | 快速响应 | 高频 Zig-zag |
| 0.8s | 平衡 | 默认 |
| 1.0-1.5s | 防重复触发 | 宽 Zig-zag |

**验证方法**：
- 统计每次 Zig-zag 半周期的峰值触发次数
- 应该 ≈ 1

### 3.3 迟滞调优

**症状**：
- 在平台期振荡
- 过峰不稳定

**调优参数**：
```python
tracking.hysteresis_fraction = 0.08  # 默认值
```

**调优指南**：
| 迟滞 | 效果 | 适用场景 |
|------|------|----------|
| 0.05 | 敏感、可能振荡 | 稳定信号 |
| 0.08 | 平衡 | 默认 |
| 0.12-0.15 | 稳定、保守 | 噪声环境 |

**验证方法**：
- 查看 Signal View 的峰值标记
- 不应该在峰值的"平台"处多次触发

## 4. 拟合与估计层调优

### 4.1 历史容量调优

**症状**：
- 转弯响应慢（case2）
- 方向估计不稳定

**调优参数**：
```python
tracking.weighted_fitter_capacity = 16  # 默认值
```

**调优指南**：
| 容量 | 效果 | 适用场景 |
|------|------|----------|
| 6-8 | 快速适应转弯 | 频繁转弯 |
| 12-16 | 平衡 | 默认 |
| 20+ | 稳定估计 | 直线电缆 |

**验证方法**：
- case2：转弯响应时间应 < 2 次穿缆
- 其他场景：航向误差应 < 5°

### 4.2 遗忘因子调优

**症状**：
- 新峰值影响太小（转弯慢）
- 旧数据拖累估计

**调优参数**：
```python
# 在 WeightedSlidingWindowFitter 中
# 遗忘因子硬编码为指数权重
```

**调优建议**：
修改权重计算方式：

```python
# 指数遗忘（默认）
weights = forgetting_factor ** np.arange(n-1, -1, -1)

# 更激进遗忘
weights = (forgetting_factor ** 1.5) ** np.arange(n-1, -1, -1)

# 更保守遗忘
weights = (forgetting_factor ** 0.8) ** np.arange(n-1, -1, -1)
```

### 4.3 拟合残差容差调优

**症状**：
- 拟合频繁被拒绝
- 航向估计不稳定

**调优参数**：
```python
tracking.fit_acceptance_residual_m = 15.0  # 默认值
tracking.fit_reject_heading_delta_deg = 45.0  # 默认值
```

**调优指南**：

**拟合拒绝过多**：
- 提高 `fit_acceptance_residual_m`（如 20m）
- 提高 `fit_reject_heading_delta_deg`（如 60°）
- 降低 `fit_reject_confidence_threshold`（如 0.3）

**拟合接受过多（离群点）**：
- 降低 `fit_acceptance_residual_m`（如 10m）
- 降低 `fit_reject_heading_delta_deg`（如 30°）
- 提高 `fit_reject_confidence_threshold`（如 0.5）

## 5. 控制决策层调优

### 5.1 置信度门限调优

**症状**：
- 难以进入 HOLD 模式
- 过早进入 HOLD 导致丢失

**调优参数**：
```python
tracking.high_confidence_threshold = 0.7  # 默认值
tracking.low_confidence_threshold = 0.3   # 默认值
```

**调优指南**：
| high_threshold | low_threshold | 效果 |
|----------------|---------------|------|
| 0.9 | 0.5 | 保守：难进入 HOLD |
| 0.7 | 0.3 | 平衡：默认 |
| 0.5 | 0.2 | 激进：易进入 HOLD |

**验证方法**：
- case1 应能稳定进入 HOLD
- case3 应在 SEARCH/APPROACH 间振荡

### 5.2 Zig-zag 宽度调优

**症状**：
- 宽度过小 → 频繁丢失
- 宽度过大 → 响应慢、精度低

**调优参数**：
```python
tracking.min_zigzag_width_m = 2.5  # 默认值
tracking.max_zigzag_width_m = 9.0   # 默认值
tracking.zigzag_width_gain_m_per_nt = 0.12  # 默认值
```

**调优指南**：

**紧凑跟踪**（直线、高 SNR）：
```python
min_zigzag_width_m = 2.0
max_zigzag_width_m = 6.0
zigzag_width_gain_m_per_nt = 0.08
```

**宽松搜索**（转弯、低 SNR）：
```python
min_zigzag_width_m = 3.5
max_zigzag_width_m = 12.0
zigzag_width_gain_m_per_nt = 0.15
```

**验证方法**：
- 查看 Top-down View 的 Zig-zag 模式
- 应该与电缆宽度相匹配

### 5.3 超时参数调优

**症状**：
- 过快声明丢失
- 过慢恢复搜索

**调优参数**：
```python
tracking.lost_timeout_s = 4.0  # 默认值
tracking.guidance_memory_timeout_s = 3.0  # 默认值
```

**调优指南**：
| lost_timeout | 效果 | 适用场景 |
|--------------|------|----------|
| 2.0s | 快速丢失、频繁搜索 | 高动态 |
| 4.0s | 平衡 | 默认 |
| 6.0s+ | 慢速丢失、长记忆 | 稳定环境 |

## 6. 系统级调优

### 6.1 性能指标平衡

| 指标 | 权衡 | 调优方向 |
|------|------|----------|
| 航向精度 vs. 响应速度 | 提高精度 → 降低速度 | 调整拟合容量、遗忘因子 |
| 覆盖率 vs. 跟踪精度 | 提高覆盖率 → 降低精度 | 调整 Zig-zag 宽度 |
| 鲁棒性 vs. 灵敏度 | 提高鲁棒性 → 降低灵敏度 | 调整峰值阈值、迟滞 |

### 6.2 多场景权衡

**场景 A（直线、高 SNR）**：
- 优化精度：小宽度、高置信度门限
- 优化拟合：大容量、小遗忘因子

**场景 B（转弯、中 SNR）**：
- 优化响应：中等宽度、中等门限
- 优化拟合：小容量、大遗忘因子

**场景 C（噪声、低 SNR）**：
- 优化鲁棒性：大宽度、低门限
- 优化峰值：高阈值、大迟滞

## 7. 常见问题诊断

### 7.1 问题诊断树

```
性能不佳
    │
    ├─► 峰值检测失败？
    │      │
    │      ├─► 是 → 检查信号处理（带通、RMS）
    │      │         检查峰值阈值、迟滞
    │      │
    │      └─► 否 → 继续
    │
    ├─► 拟合残差大？
    │      │
    │      ├─► 是 → 检查容量、遗忘因子
    │      │         检查离群点拒绝
    │      │
    │      └─► 否 → 继续
    │
    ├─► 模式振荡？
    │      │
    │      ├─► 是 → 检查置信度门限
    │      │         检查超时参数
    │      │
    │      └─► 否 → 继续
    │
    └─► 轨迹质量差？
           │
           ├─► 是 → 检查 Zig-zag 宽度
           │         检查速度、转弯半径
           │
           └─► 否 → 可能已是最优
```

### 7.2 症状-原因-解决方案表

| 症状 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无峰值触发 | 带通失效 | 检查采样率、频率匹配 |
| 峰值过多 | 阈值过低 | 提高 min_peak_strength_nt |
| 转弯响应慢 | 遗忘因子大 | 降低容量、增加遗忘 |
| 航向跳变 | 拟合不稳定 | 增加容量、检查离群点 |
| 频繁丢失 | 宽度过小 | 提高 min_zigzag_width_m |
| 难进 HOLD | 门限过高 | 降低 high_confidence_threshold |
| 轨迹振荡 | 灵敏度高 | 提高迟滞、降低阈值 |

## 10. 调优最佳实践

1. **一次只改一个参数**：隔离效果
2. **小步迭代**：每次变化 < 20%
3. **记录一切**：配置、结果、分析
4. **自动化验证**：脚本化重复工作
5. **保持基线**：随时可以回退
6. **多场景验证**：一个场景的优化不应破坏其他场景
7. **理解权衡**：没有"最优"，只有"最合适"

## 11. 总结

调优是科学的艺术：

- **科学**：系统化方法、可控实验、数据驱动

**调优 = 观察 + 假设 + 验证 + 迭代**
