# 可视化体系

> **定位**：本系统的离线可视化与成果展示链路（重构 Phase 2V/2W）。统一入口 `tools/visualize.py`，
> 所有图像/报告输出到 `results/<时间戳>/`。
> **设计原则**：GUI/Logic 严格分离——`viz/figures.py` 是**唯一** import matplotlib 的模块，其余（`metrics.py`/`recorder.py`/`report.py`/`baseline.py`）均为纯数据/纯文本，保证指标有单一事实来源。

---

## 1. 架构总览

```
tools/visualize.py  (CLI 单一入口)
        │
        ▼
viz.simulate_case(case, deployment_mode, max_steps)
        │   → RunRecord（逐帧列存数据，唯一数据契约）
        ▼
viz.compute_health_metrics(record) → HealthMetrics（纯函数指标）
        │
        ├── viz.render_run(record, metrics, fig_dir)   → overview.png + detail.png
        ├── viz.save_run_report(metrics, figs, path)   → report.md（含 JSON）
        ├── viz.render_showcase(metrics_list, path)    → showcase.png
        ├── viz.save_showcase_report(...)              → showcase.md
        ├── viz.render_progress(deltas, path)          → progress.png
        └── viz.save_progress_report(...)              → progress.md
```

| 模块 | 职责 | 是否 import matplotlib |
|---|---|---|
| [`viz/recorder.py`](../src/auv_mag_tracking/viz/recorder.py) | 唯一离线仿真驱动 → `RunRecord`（列存通道，可存 `.npz`） | 否（纯数据） |
| [`viz/metrics.py`](../src/auv_mag_tracking/viz/metrics.py) | `compute_health_metrics` / `health_score` / 进度对照（纯函数） | 否 |
| [`viz/baseline.py`](../src/auv_mag_tracking/viz/baseline.py) | 固化里程碑基线常量（`PRE_2G` 等），支持 before/after 复现 | 否 |
| [`viz/report.py`](../src/auv_mag_tracking/viz/report.py) | Markdown 报告组装（含自动分析、JSON 落盘） | 否 |
| [`viz/figures.py`](../src/auv_mag_tracking/viz/figures.py) | 学术风格静态图渲染（IEEE/HKU、衬线、冷暖语义配色） | **是（唯一）** |

## 2. 如何生成图像（命令清单）

> 前置：`python3` 需能 import `auv_mag_tracking`（CLI 已自动把 `src/` 加入 `sys.path`）。

```bash
# 单个场景：完整报告（overview + detail 图 + report.md + record.npz）
python3 tools/visualize.py --case case5

# 全部核心场景（case1..6）+ 跨 case 成果汇总（showcase）
python3 tools/visualize.py --all

> 当前 `--all` 输出 `case1`–`case6`；`case6` 会单独落盘到 `results/<timestamp>/case6/`。

# 后续转弯变种（case1v..case6v）+ 跨变种成果汇总
python3 tools/visualize.py --variants

# 长时压力测试（例如把变种统一拉长到 900 s）
python3 tools/visualize.py --variants --duration-s 900

# 光滑迷宫往返长电缆压力测试（有声呐 + 有声呐后失效 + 无声呐）
python3 tools/visualize.py --maze

> 当前 `--maze` 输出 `case_maze_sonar`、`case_maze_sonar_dropout` 和 `case_maze_no_sonar`。

# before/after 进度对照（对照 viz/baseline.py 固化基线，生成 progress.png/md）
python3 tools/visualize.py --progress

# 部署模式（禁用名义路由先验）
python3 tools/visualize.py --all --deployment

# 限制步数（快速冒烟）
python3 tools/visualize.py --case case1 --max-steps 500

# 指定输出目录（默认 results/<时间戳>/）
python3 tools/visualize.py --all --outdir results/my_run

# 实时仪表盘（matplotlib 动画，非离线落盘）
python3 tools/visualize.py --live --case case1
```

## 3. 输出在哪里？目录结构

> **重要**：`results/` 已被 `.gitignore` 忽略（见 [`.gitignore`](../.gitignore) 第 18 行），
> 因此图像**不会进版本库**；这也是 `viz/baseline.py` 把里程碑数字固化为代码常量的原因——
> 让进度对照能从干净 checkout 复现，不依赖本地 `results/`。

```
results/
└── 20260624_153012/              # 运行时间戳（每次运行一个新目录）
    ├── case1/
    │   ├── figures/
    │   │   ├── case1_overview.png    # 4 面板速览图（论文/幻灯片用，粗边框）
    │   │   └── case1_detail.png      # 9 面板诊断仪表盘
    │   ├── record.npz                # 逐帧数值通道（二进制，可复算）
    │   └── report.md                 # 单 case 报告（图 + 指标表 + 自动分析 + JSON）
    ├── case2/ ... case6/            # 同上结构
    ├── showcase.png                  # 跨 case 成果汇总图（--all）
    ├── showcase.md                   # 跨 case 场景矩阵报告
    ├── progress.png                  # before/after 进度图（--progress）
    └── progress.md                   # 逐 case 进度对照矩阵 + headline
```

## 4. 如何查看这些图像

| 方式 | 操作 |
|---|---|
| macOS 预览 | `open results/<时间戳>/case5/figures/case5_overview.png` |
| 打开整个目录 | `open results/<时间戳>/` |
| 阅读报告（含内嵌图引用） | 用任意 Markdown 阅读器打开 `report.md` / `showcase.md` / `progress.md`（图与 md 同目录，相对路径引用） |
| 程序化复算 | `np.load("results/<时间戳>/case5/record.npz", allow_pickle=True)` 读取逐帧通道 |

> 报告里的图片是相对路径引用（如 `![overview](figures/case1_overview.png)`），
> 所以请保持 `report.md` 与其 `figures/` 在同一目录，移动时整目录一起移动。

## 5. 两层图像各自画什么

### 5.1 overview（4 面板，幻灯片/论文用）
- 轨迹俯视图（真值缆 + AUV 轨迹 + 拟合中心线，冷暖语义配色）
- 航向误差时间序列（含 15° 硬约束参考线）
- FSM 状态占用带（search/align/track/emergency 色块）
- 关键指标摘要（health、平均误差、TRACK 占比、切换数）

### 5.2 detail（9 面板，诊断用）
- 在 overview 基础上扩展：SNR 时序、置信度、拟合残差、横偏、磁峰事件、导航源占比等。

### 5.3 showcase（跨 case）
- 把 case1–5 的 health / 误差 / TRACK% / 切换数横向并排，一眼看出哪个场景是瓶颈。

### 5.4 progress（before/after）
- 对照 `viz/baseline.py` 固化基线，逐字段给"改善/持平/回退"结论（带容差带，避免浮点抖动误判），
  并汇总 headline（如"FSM 切换风暴 total X→Y，N/M case 达 15° 硬约束"）。

## 6. 报告里的健康分（health_score）口径

`health_score`（[`viz/metrics.py`](../src/auv_mag_tracking/viz/metrics.py)）是 0–100 的加权总分（越高越好）：

| 维度 | 权重 | 满分条件 | 零分条件 |
|---|---|---|---|
| fused-heading 平均误差 | 10 | 0° | ≥ 35° |
| <15° 好估计占比 | 5 | 100% | 0% |
| TRACK 段车辆航向误差 | 10 | 0° | ≥ 25° |
| TRACK 段横向距离 | 20 | 0 m | ≥ 12 m |
| TRACK_ACTIVE 占用 | 10 | ≥ 30% | 0% |
| 模式稳定性 | 10 | ≤ 6 次切换 | ≥ 30 次 |
| 最终横向距离 | 10 | 0 m | ≥ 18 m |
| route/endpoint 完成度 | 25 | endpoint 验收场景：`route>=100%` 且 endpoint；定时基准场景：不扣分 | endpoint 验收场景 0% route |

> 该口径是单一事实来源：实时仪表盘与离线报告共用同一函数，不存在"两套指标漂移"。当前版本刻意降低了全程 fused-heading 均值的权重，并提高 TRACK 段闭环指标权重，避免 `case6` 这类强弯场景被单一角度口径误判为完全失败。

## 7. 二次开发：新增一张图 / 一个指标

- **新增指标**：在 `viz/metrics.py` 的 `HealthMetrics` 加字段并在 `compute_health_metrics` 计算（纯函数，无 I/O）；
  报告若要展示，在 `viz/report.py` 的表格里加一行。
- **新增图**：在 `viz/figures.py` 写 `render_xxx(record, metrics, out_path)`，**只在此模块** import matplotlib；
  在 `render_run` 或 CLI 里挂上。
- **新增通道**：在 `viz/recorder.py` 的 `_NUMERIC_CHANNELS` 加列名并在记录循环里写入，自动进入 `.npz`。

> 详见 [`11_二次开发指南.md`](11_二次开发指南.md)。
