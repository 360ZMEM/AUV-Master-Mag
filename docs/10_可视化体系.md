# 可视化体系

> **定位**：本系统的离线可视化与成果展示链路（重构 Phase 2V/2W）。统一入口 `tools/visualize.py`，
> 所有图像/报告输出到 `results/<时间戳>/`。
> **设计原则**：GUI/Logic 严格分离——`viz/figures.py` 是**唯一** import matplotlib 的模块，其余（`metrics.py`/`recorder.py`/`report.py`/`baseline.py`）均为纯数据/纯文本，保证指标有单一事实来源。

---

## 1. 架构总览

```
tools/visualize.py  (CLI 单一入口)
        │
        ▼
viz.simulate_case(case, deployment_mode, max_steps)
        │   → RunRecord（逐帧列存数据，唯一数据契约）
        ▼
viz.compute_health_metrics(record) → HealthMetrics（纯函数指标）
        │
        ├── viz.render_run(record, metrics, fig_dir, paper_panels=True)
        │     → overview.png + detail.png + selector_sync.png  （工程整体大图）
        │     → figures/paper/<slug>.{png,pdf}                  （论文单面板小图）
        ├── viz.save_run_report(metrics, figs, path)   → report.md（含 JSON）
        ├── viz.render_showcase(metrics_list, path)    → showcase.png
        ├── viz.render_paper_showcase_panels(...)      → paper/showcase_*.{png,pdf}
        ├── viz.save_showcase_report(...)              → showcase.md
        ├── viz.render_progress(deltas, path)          → progress.png
        ├── viz.render_paper_progress_panels(...)      → paper/progress_*.{png,pdf}
        └── viz.save_progress_report(...)              → progress.md
```

| 模块 | 职责 | 是否 import matplotlib |
|---|---|---|
| [`viz/recorder.py`](../src/auv_mag_tracking/viz/recorder.py) | 唯一离线仿真驱动 → `RunRecord`（列存通道，可存 `.npz`） | 否（纯数据） |
| [`viz/metrics.py`](../src/auv_mag_tracking/viz/metrics.py) | `compute_health_metrics` / `health_score` / 进度对照（纯函数） | 否 |
| [`viz/baseline.py`](../src/auv_mag_tracking/viz/baseline.py) | 固化里程碑基线常量（`PRE_2G` 等），支持 before/after 复现 | 否 |
| [`viz/report.py`](../src/auv_mag_tracking/viz/report.py) | Markdown 报告组装（含自动分析、JSON 落盘） | 否 |
| [`viz/figures.py`](../src/auv_mag_tracking/viz/figures.py) | 学术风格静态图渲染（IEEE/HKU、衬线、冷暖语义配色） | **是（唯一）** |

## 2. 如何生成图像（命令清单）

> 前置：需使用带 matplotlib 的 Python 环境（本仓库默认 `python` 指向 miniconda3）。

```bash
# 单个场景：完整报告（overview + detail + selector_sync 图 + report.md + record.npz）
python tools/visualize.py --case case5

# 全部核心场景（case1..6）+ 跨 case 成果汇总（showcase）
python tools/visualize.py --all

> 当前 `--all` 输出 `case1`–`case6`；`case6` 会单独落盘到 `results/<timestamp>/case6/`。

# 后续转弯变种（case1v..case6v）+ 跨变种成果汇总
python tools/visualize.py --variants

# 长时压力测试（例如把变种统一拉长到 900 s）
python tools/visualize.py --variants --duration-s 900

# 光滑迷宫往返长电缆压力测试（有声呐 + 有声呐后失效 + 稀疏声呐 + 无声呐）
python tools/visualize.py --maze

> 当前 `--maze` 输出 `case_maze_sonar`、`case_maze_sonar_dropout`、`case_maze_sparse_sonar` 和 `case_maze_no_sonar`。

# before/after 进度对照（对照 viz/baseline.py 固化基线，生成 progress.png/md）
python tools/visualize.py --progress

# 部署模式（禁用名义路由先验）
python tools/visualize.py --all --deployment

# 限制步数（快速冒烟）
python tools/visualize.py --case case1 --max-steps 500

# 指定输出目录（默认 results/<时间戳>/）
python tools/visualize.py --all --outdir results/my_run

# 关闭论文小图（仅保留工程大图）
python tools/visualize.py --all --no-paper-figures

# 实时仪表盘（matplotlib 动画，非离线落盘）
python tools/visualize.py --live --case case1
```

## 5. 三层图像 + 论文小图各自画什么

### 5.1 overview（4 面板，投影/汇报用，大整体图）
- 轨迹俯视图（真值缆 + AUV 轨迹 + 拟合中心线，冷暖语义配色）
- 航向误差时间序列（含 15° 硬约束参考线 + FSM 状态着色背景）
- 横向距离到真值缆 + mean 参考线
- 融合置信度 + safe-lock 阴影 + 声呐/磁制导贡献占比

### 5.2 detail（12 面板，工程诊断用）
- 在 overview 基础上扩展：真值/融合/线拟合三路航向、SNR、拟合协方差 λ⊥、磁横偏、磁强度峰值、FSM 阶梯、lookahead feed 拒绝原因、feed 余量、zig-zag probe cycle、route progress、紧凑轨迹等。

### 5.3 selector_sync（4 面板，shadow 候选选择器诊断）
- Route progress + FSM 着色（用于 Progress Guard）。
- Progress rate + readiness 阈值。
- 假设/验证/dual/aligned 闸门叠加，对比 magnetic_lookahead_feed_allowed。
- 验证/dual/progress/aligned 多类拒绝原因码叠加 + progress proxy / route-bound proxy 有效性。

### 5.4 showcase（4 面板，跨 case 大整体图）
- 把 case1–6 或 case1v–6v 或 maze 系列的 fused/TRACK heading、FSM 状态占比、声呐+磁贡献、健康得分横向并排。

### 5.5 progress（4 面板，before/after 大整体图）
- 对照 `viz/baseline.py` 固化基线，逐字段给"改善/持平/回退"结论，并汇总 headline。

### 5.6 论文单面板小图（IEEE 双栏 ~3.5 in，PNG + PDF）
- 默认与 5.1–5.5 一起生成，落到 `figures/paper/` 与 `results/<ts>/paper/`。
- 字号 8–9 pt、线粗 1.0、刻度细线、紧 bbox、衬线字体（Times / DejaVu Serif）。
- 共 20 张 run-level + 4 张 showcase + 4 张 progress，完整 slug 表见 [23_论文图清单](23_论文图清单.md)。
- 单面板图与整体大图共用 painter（`_draw_xxx(ax, ...)` 纯函数），因此面板内容一致、不会双线。

## 6. 报告里的健康分（health_score）口径

`health_score`（[`viz/metrics.py`](../src/auv_mag_tracking/viz/metrics.py)）是 0–100 的加权总分（越高越好）：

| 维度 | 权重 | 满分条件 | 零分条件 |
|---|---|---|---|
| fused-heading 平均误差 | 10 | 0° | ≥ 35° |
| <15° 好估计占比 | 5 | 100% | 0% |
| TRACK 段车辆航向误差 | 10 | 0° | ≥ 25° |
| TRACK 段横向距离 | 20 | 0 m | ≥ 12 m |
| TRACK_ACTIVE 占用 | 10 | ≥ 30% | 0% |
| 模式稳定性 | 10 | ≤ 6 次切换 | ≥ 30 次 |
| 最终横向距离 | 10 | 0 m | ≥ 18 m |
| route/endpoint 完成度 | 25 | endpoint 验收场景：`route>=100%` 且 endpoint；定时基准场景：不扣分 | endpoint 验收场景 0% route |

> 该口径是单一事实来源：实时仪表盘与离线报告共用同一函数，不存在"两套指标漂移"。当前版本刻意降低了全程 fused-heading 均值的权重，并提高 TRACK 段闭环指标权重，避免 `case6` 这类强弯场景被单一角度口径误判为完全失败。

## 7. 二次开发：新增一张图 / 一个指标

- **新增指标**：在 `viz/metrics.py` 的 `HealthMetrics` 加字段并在 `compute_health_metrics` 计算（纯函数，无 I/O）；
  报告若要展示，在 `viz/report.py` 的表格里加一行。
- **新增图（painter 模式）**：
  1. 在 `viz/figures.py` 写 painter `_draw_xxx(ax, record, metrics)`，**只在 figures.py 内** import matplotlib。
  2. 在整体大图（`render_overview` / `render_detail` / `render_selector_sync` / `render_showcase` / `render_progress`）中通过 `fig.add_subplot()` + 调用 `_draw_xxx(ax, ...)` + `ax.set_title("(a/1/i) ...")` 接入。
  3. 若该面板需要论文小图，把 `(slug, lambda ax,r,m: (_draw_xxx(ax,r,m), ax.set_title("..."))[0], figsize, tier)` 加入 `_PAPER_RUN_PANELS` 注册表；showcase/progress 同理放入 `_PAPER_SHOWCASE_PANELS` 或 `_PROGRESS_PANELS`。
  4. CLI 已自动调用 `render_paper_run_panels` / `render_paper_showcase_panels` / `render_paper_progress_panels`，无需改 `tools/visualize.py`。
- **新增通道**：在 `viz/recorder.py` 的 `_NUMERIC_CHANNELS` 加列名并在记录循环里写入，自动进入 `.npz`。

> 详见 [`11_二次开发指南.md`](11_二次开发指南.md) 与 [`23_论文图清单.md`](23_论文图清单.md)。

### 9.5 maze 几何验收指标

迷宫场景不能只看 `route_completion_ratio` 和 `endpoint_completed`。最近点投影可能从当前 lane 跳到远端 lane，造成“route 接近 100%，但 AUV 实际抄近路”的假阳性。因此 `HealthMetrics` 额外输出以下字段：

| 指标 | 含义 |
| --- | --- |
| `p99_cross_track_m` | 99 分位横偏，用于暴露少量极端脱轨帧 |
| `route_progress_backward_fraction` | route progress 单步后退超过 1m 的帧占比 |
| `route_progress_max_jump_m` | 相邻帧 route progress 最大正向跳变 |
| `route_progress_large_jump_count` | route progress 单步跳变超过 25m 的次数 |
| `lane_shortcut_indicator` | 是否检测到大 route jump；`yes` 表示存在跨 lane / projection jump 风险 |
| `maze_geometry_passed` | maze 专用几何验收；route jump、route fail 或 TRACK vehicle heading fail 任一成立则为 `no` |

当前 `maze_geometry_passed` 的判定逻辑：

- `route_progress_large_jump_count > 0`：几何失败。即使 endpoint 完成，也认为有 lane shortcut / projection jump。
- endpoint goal 启用但 `route_completion_ratio < 95%`：任务进度失败。
- TRACK 段车辆真实航向误差 `track_mean_vehicle_heading_error_deg > 45°`：车辆没有沿任务路线推进。

这组指标只用于 maze 几何验收和报告解释，不替代普通 `case1`-`case6` 的健康分主口径。

### 10.1 maze geometry 自动分析行

`report.md` 在 `case_maze*` 场景中还会输出 maze geometry 判定：

```text
- MAZE GEOMETRY PASS: no large route-progress shortcut detected.
- MAZE GEOMETRY FAIL: route progress jumped by X.X m, indicating a lane shortcut / projection jump.
- MAZE GEOMETRY FAIL: task progress or vehicle heading failed maze acceptance.
```

阅读顺序建议：

1. 先看 `Route completion` 和 `Endpoint completed`，判断任务是否到终点。
2. 再看 `Route progress max jump`、`Route progress large jumps`、`Lane shortcut indicator`，判断是否抄近路。
3. 最后看 `TRACK mean vehicle heading error` 和 `TRACK mean cross-track`，判断 AUV 是否真的沿缆推进。

因此，`endpoint=yes` 但 `Maze geometry passed=no` 的结果不能作为 maze 成功；这通常意味着车辆通过最近点投影跳变"看起来到终点"，但几何轨迹不合格。
