# 任务状态机方法论

> **架构沿革**：早期版本用六节点优先级行为树（`behavior_tree.py`）做策略决策。
> 重构后该模块已删除，策略层收敛为一个**三态有限状态机**
> （[`mission_manager.py`](../src/auv_mag_tracking/mission_manager.py)）。本文描述
> 现行架构；行为树时代的设计讨论见 [`dev_log/`](dev_log/)。

## 1. 为什么是三态 FSM 而非行为树

行为树用六个优先级节点（SafeLock / Turn / Hold / Approach / Lost / Search）覆盖
所有情境。实践中这套结构有两个问题：

1. **节点边界与物理阶段不对齐**：Turn/Hold/Approach 三者只是"同一个跟踪阶段"的
   置信度切片，却被拆成三个节点，转移条件互相耦合，难以推理。
2. **优先级抢占引发抖振**：相邻节点的触发条件在噪声下反复翻转（SEARCH↔LOST），
   产生大量无意义的模式切换。

电缆跟踪任务的物理阶段其实只有三个——**找到、对齐、跟踪**。把策略层收敛成与这三个
阶段一一对应的状态，转移判据清晰、可去抖、可观测：

```
SEARCH_ZIGZAG  → 在先验走廊里之字形横扫，直到感知到电缆
LOCK_ALIGN     → 减速，让中心线拟合在过线处收敛
TRACK_ACTIVE   → 声呐/磁协同的稳态跟踪
EMERGENCY_SURFACE（终态）→ 双盲，仅记录（速度→0 + 置位标志）
```

所有阈值集中在 [`MissionThresholds`](../src/auv_mag_tracking/mission_manager.py)，
FSM 本身只持有去抖所需的最小计数器。

## 2. 状态与职责

| 状态 | 触发进入 | 速度因子 | guidance_source | 控制层动作 |
|------|----------|----------|-----------------|------------|
| **SEARCH_ZIGZAG** | 初始 / 信号丢失退回 | 1.0 | `SEARCH` | 沿先验走廊做满宽之字形横扫 |
| **LOCK_ALIGN** | 信号锁存 `lock_streak` 步 | 0.5 | `MAGNETIC[_PEAK]` | 减速，继续横切让拟合收敛 |
| **TRACK_ACTIVE** | 拟合收敛 `track_streak` 步 | 1.0 | `MAGNETIC[_PEAK]` | 压线跟踪（声呐＞磁＞先验） |
| **EMERGENCY_SURFACE** | TRACK 置信度持续低于地板 | 0.0 | `EMERGENCY` | 速度归零 + emergency_flag |

> 控制层把 `guidance_source` 中的 `_PEAK` 后缀仅作诊断标注；纵向基准始终来自先验
> 切向，磁/声呐只负责横向回缆（见 [传感器融合方法论](04_传感器融合方法论.md)）。

## 3. 状态转移判据

### 3.1 信号存在判定（带迟滞锁存）

之字形横切让电缆磁场每次过线都"升–峰–落"，单一门限会让"有信号"判定反复穿越。
故采用**双门限迟滞**（[`_signal_present`](../src/auv_mag_tracking/mission_manager.py)）：

- **上升沿锁存**：`mag_strength ≥ mag_lock_threshold_nT`（50 nT）或
  `sonar_confidence ≥ sonar_confidence_threshold`（0.6）→ 置 `signal_latched`
- **下降沿解除**：磁强跌破 `mag_release_threshold_nT`（25 nT）**且**声呐也不达标 →
  清 `signal_latched`

锁存带（50 / 25 nT）跨越自然峰谷保持稳定。

### 3.2 信号丢失判定（时效而非计数）

用"距上次检测的时长"代替短促的连续漏检计数
（[`_signal_lost`](../src/auv_mag_tracking/mission_manager.py)）：只要一个完整扫描
周期（`signal_hold_s = 12 s`，≈ 一次 zig-zag 扫描）内出现过信号，就视为仍在跟踪，
避免每个扫描谷底都误判丢失而退态。

### 3.3 拟合收敛判定（EKF P_yy 代理）

LOCK_ALIGN → TRACK_ACTIVE 需要"拟合已收敛"的证据。系统无 EKF，故用 PCA 加权散布
协方差的**较小特征值**（垂直方向散布）作为 EKF `P_yy` 的代理
（[`_fit_converged`](../src/auv_mag_tracking/mission_manager.py)）。三条件同时满足：

```
eig_perp < cov_perp_converged_m2 (1.0)          # 拟合垂直散布足够小
|yaw_error_deg| < yaw_err_converged_deg (5.0)   # 航向误差达标
confidence ≥ track_confidence_required (0.65)   # 置信度足够
```

### 3.4 转移图

```
        ┌──────────────────┐
        │  SEARCH_ZIGZAG   │◄────────────────┐
        └────────┬─────────┘                 │ signal_lost
   signal_latched │ 连续 lock_streak (5) 步    │
        ┌────────▼─────────┐                 │
        │   LOCK_ALIGN     │─────────────────┘
        └────────┬─────────┘◄───────────┐
   fit_converged │ 连续 track_streak (5) │ signal_lost
        ┌────────▼─────────┐            │ (置信度未跌破地板)
        │  TRACK_ACTIVE    │────────────┘
        └────────┬─────────┘
   confidence < system_confidence_floor (0.1)
        持续 emergency_hold_s (5 s)
        ┌────────▼─────────┐
        │EMERGENCY_SURFACE │ (终态，不可逆)
        └──────────────────┘
```

关键不对称：TRACK_ACTIVE 在信号丢失时**退回 LOCK_ALIGN**（而非 SEARCH），保留已建立
的中心线；只有置信度持续跌破地板才进入不可逆的 EMERGENCY 终态。

## 4. 阈值参考

| 阈值 | 默认值 | 作用 |
|------|--------|------|
| `mag_lock_threshold_nT` | 50.0 | 信号锁存上升沿 |
| `mag_release_threshold_nT` | 25.0 | 信号解除下降沿（迟滞带下界） |
| `sonar_confidence_threshold` | 0.6 | 声呐信号达标门限 |
| `lock_streak_required` | 5 | SEARCH→LOCK 所需连续锁存步数 |
| `signal_hold_s` | 12.0 | 丢失判定时效（≈一次扫描周期） |
| `align_speed_factor` | 0.5 | LOCK_ALIGN 减速因子 |
| `cov_perp_converged_m2` | 1.0 | 拟合垂直散布收敛门限（P_yy 代理） |
| `yaw_err_converged_deg` | 5.0 | 航向误差收敛门限 |
| `track_confidence_required` | 0.65 | 进入跟踪所需置信度 |
| `track_streak_required` | 5 | LOCK→TRACK 所需连续收敛步数 |
| `system_confidence_floor` | 0.1 | 触发应急的置信度地板 |
| `emergency_hold_s` | 5.0 | 持续低置信度多久后应急 |

## 5. 接口契约

```python
@dataclass
class MissionInput:          # 由 PerceptionState + Pose 投影而来
    time_s: float
    mag_strength_nT: float
    sonar_confidence: float
    confidence: float
    fused_heading_deg: Optional[float]
    yaw_error_deg: Optional[float]
    fit_covariance_xy_m2: Optional[np.ndarray]
    peak_detected: bool

@dataclass
class MissionDecision:       # 输出给控制器
    state: MissionState
    speed_factor: float
    guidance_source: str
    emergency_flag: bool = False
```

控制器 [`controller.py`](../src/auv_mag_tracking/controller.py) 每步调用
`MissionManager.update(mission_input)`，据 `state` 选择横切角与压线策略，据
`speed_factor` 设定速度。任务层**不**直接产生航向指令——它只决定"处于哪个阶段"，
具体的横向回缆由控制层按 `声呐 > 磁 > 先验` 优先级执行。

## 6. 可观测性

`mode_switches`（统一可视化体系 [viz](../src/auv_mag_tracking/viz/) 输出）是 FSM
健康度的核心指标：重构后全 case 切换数收敛到 2–4 次（行为树时代频繁抖振）。典型序列：

- **case1（直线）**：`SEARCH → LOCK_ALIGN → TRACK_ACTIVE`（一次性收敛）
- **case2（转弯）**：进入 TRACK 后弯段拟合短暂退回 LOCK_ALIGN，再收敛

异常信号：长期停留 SEARCH（信号始终不锁存）、TRACK↔LOCK 频繁往返（收敛判据临界抖动）。

## 7. 小结

三态 FSM 的设计精神：**状态与物理阶段一一对应，转移判据可去抖、可观测**。

- **找到（SEARCH）**：迟滞锁存确认信号存在
- **对齐（LOCK）**：P_yy 代理确认拟合收敛
- **跟踪（TRACK）**：丢失退回 LOCK 而非从头搜索，置信度地板兜底应急

与硬编码状态机和优先级行为树相比，三态 FSM 在保持表达力的同时大幅降低了切换抖振，
且全部阈值集中可配、便于人工调参。
