# 迷宫场景算法重构方案

> 分支：`feature/maze-algorithm-refactor`
>
> 基准尺寸固定为当前 2×：长边 600 m、lane 间距 200 m、U-turn 半径 60 m。后续不再通过 4×/8× 环境放大回避算法问题。

## 1. 问题定义

验收标准：`case_maze_sonar` / `case_maze_no_sonar` 在无强路由先验下持续跟踪到电缆终点，并由 `stop_reason=endpoint` 结束。

当前实测：

| case | route progress | stop_reason | 判断 |
|---|---:|---|---|
| `case_maze_sonar` | 38.2% | `duration` | 未通过 |
| `case_maze_no_sonar` | 0.3% | `duration` | 未完成初始捕获 |

注意：`route_progress` 可能因最近点投影跳到远端而产生假阳性；最终必须同时检查 `endpoint_completed=1`、`stop_reason=endpoint` 和 endpoint 横向容差。

## 2. 已证伪的最小修正

以下改动均不保留为主线代码：

| 尝试 | 结果 | 结论 |
|---|---|---|
| fitter 观测年龄窗 | sonar progress 38.2% → 17.0% | 硬删除旧点破坏初始捕获 |
| fitter 时间半衰权重 | 最好仍未 endpoint | 只改变误差均值，不解决闭环 |
| 声呐航向与 PCA 航向门控 | 17.0%–37.9% | 改变状态机节奏但不能建立持续局部轨迹 |
| 声呐在线时持续接管航向 | 38.1%，未 endpoint | 单步航向正确不等于路径状态稳定 |
| TRACK zig-zag / cross-track 参数矩阵 | 最好 61.9% 但距路由 1625 m | route progress 为投影假阳性 |
| 4× 等比放大 | 约 37.8% | 尺寸放大不是解法 |

## 3. 真实根因

当前感知只有 `WeightedSlidingWindowFitter` 的线性 PCA 模型。它把峰值点和声呐点混为无时序状态的点云，输出一条直线。

迷宫场景下失败链路为：

1. SEARCH/ALIGN 阶段少量高 SNR 或高置信点进入 fitter。
2. bootstrap 后 PCA 输出局部直线，但航向可偏离 lane 真值约 17°。
3. TRACK 根据错误纵向基准前进，车辆在 lane1 中段前就漂出声呐量程。
4. 声呐脱窗后没有新的位置观测，fitter 继续冻结旧方向。
5. 控制器继续沿旧方向漂离，后续 route progress 可能只是最近点投影跳变。

这不是物理不可行域：磁场强度、声呐量程、lane 串扰都不构成首要限制。问题是估计器没有“沿时间推进的电缆状态”。

## 4. 目标架构

新增一个局部路径状态估计层，位置位于 perception 包内，职责独立于控制器：

```text
SonarReading / peak observations
        │
        ▼
LocalCableStateEstimator
        ├── 观测门控：时效、距离、航向一致性、离群点
        ├── 局部模型：全局直线 / 局部直线 / 圆弧 / 样条小窗
        ├── 时序状态：当前位置 s、切向 heading、曲率 kappa、横向残差
        └── 健康度：mature / stale / reacquire_required
        │
        ▼
PerceptionState
        ├── fused_heading_deg
        ├── estimated_cable_point_xy_m
        ├── fit_result（兼容旧接口）
        └── deployment_reacquire_required
```

控制器仍只消费 `PerceptionState`，保持 GUI/Logic 与 perception/control 分离。

## 5. 实施阶段

### Phase A：局部直线状态替换点云 PCA

目标：在直线 lane1 上消除 17° 初始偏航与旧点污染。

实现：

- 新增 `perception/local_path.py`。
- 输入声呐点与声呐航向，维护时间窗口内的局部状态。
- 先独立实现全局直线、局部直线与圆弧模型：圆弧可靠时输出 `arc`；圆弧不可靠但曲线正在转向时输出 `local_line`，只用最近小窗并按新近性加权，使切向随 AUV 前进自动转动；直线段仍保持 `line`。
- 航向观测不是只降权置信度，而是参与局部直线切向融合，避免圆弧估计不可靠时退回静态全局 PCA。
- 输出等价 `FitResult`，不改控制器契约。
- 当前已接入旁路诊断字段：`local_path_model_code`、`local_path_heading_deg`、`local_path_confidence`、`local_path_residual_m`、`local_path_radius_m`。这些字段进入 `RunRecord`。
- 加噪独立测试已覆盖 0.45 m 位置噪声和 5° 航向噪声；调优结果采用 `local_line_window=5`、`heading_blend=0.65`，典型局部切向均值误差约 4–5°。
- 主流程接入采用保守门控：默认关闭，仅在无先验且 `local_path_guidance_enabled=True` 的部署场景中允许 `LOCAL_PATH` 参与 `fused_heading_deg`；case1–6 名义场景不受影响。
- `case_maze_sonar` 接入后 route completion 从 38.2% 提升到 59.6%，但仍未达到 endpoint；`case_maze_no_sonar` 仍为 0.3%，说明无声呐下仍缺少纵向切向观测。
- 已证伪两条后续补丁路线：把 local path 直接接入 FSM 信号锁存会退化到 7.7%；在 SEARCH/ALIGN 中强行压低交叉角会退化到 14.8%。下一步应实现显式重捕获状态，而不是继续放宽 local path 门控。

验收：

- 独立单元测试先通过短曲线大转角（如 R=38m、120° 转角）与圆弧禁用场景，证明 `local_line` 会随前进改变角度。
- 旁路诊断可在 `case_maze_sonar` 中稳定产出，并与旧 PCA 航向形成可对比序列。
- 主流程接入后，`case_maze_sonar` 至少优于 38.2% 基线且不引入 case1–6 回归；当前实测为 59.6%，但尚未 endpoint。
- case1–6 指标不回归。

### Phase B：圆弧/样条小窗模型

目标：通过第一个 U-turn。

实现：

- 在局部窗口内比较直线模型与圆弧模型的 AIC/残差。
- 圆弧模型只在观测跨度和曲率显著性足够时启用。
- 输出当前位置最近点切向，而不是全局平均 PCA 方向。

验收：

- `case_maze_sonar` 通过第一个 U-turn 并进入 lane2。
- 不允许依赖仿真真值路由。

### Phase C：重捕获与无声呐变种

目标：降低声呐脱窗后的不可恢复性。

实现：

- `deployment_reacquire_required=True` 时控制器进入局部重捕获扫线，而非沿冻结 heading 远离。
- 无声呐变种先验上更困难，应先明确其传感器物理可观测性；若仅靠磁强度不能提供纵向切向，需记录为独立不可行边界，而不是强行过验收。

验收：

- `case_maze_sonar` 达到 `stop_reason=endpoint`。
- `case_maze_no_sonar` 若无法通过，必须给出物理可观测性证明和文档化边界。

## 6. 不变量

- 不向 perception 引入仿真真值。
- 不把磁横偏粗估计喂入主中心线拟合器。
- 不继续放大 2× 迷宫几何。
- 每阶段都运行 `python3 -m unittest discover -s tests` 和 `python3 tools/visualize.py --maze`。
