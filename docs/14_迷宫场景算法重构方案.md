# 迷宫场景算法重构方案

> 分支：`feature/maze-algorithm-refactor`
>
> 当前 Phase D 诊断阶段将迷宫缩回 1× prototype：长边 300 m、lane 间距 100 m、U-turn 半径 30 m。这样降低单次仿真和代表点测试成本；禁止再做大地图网格搜参。

## 1. 问题定义

验收标准：`case_maze_sonar` / `case_maze_no_sonar` 在无强路由先验下持续跟踪到电缆终点，并由 `stop_reason=endpoint` 结束。

当前实测：

| case | route progress | stop_reason | 判断 |
|---|---:|---|---|
| `case_maze_sonar` | 38.2% | `duration` | 未通过 |
| `case_maze_no_sonar` | 0.3% | `duration` | 未完成初始捕获 |

注意：`route_progress` 可能因最近点投影跳到远端而产生假阳性；最终必须同时检查 `endpoint_completed=1`、`stop_reason=endpoint` 和 endpoint 横向容差。

## 2. 已证伪的最小修正

以下改动均不保留为主线代码：

| 尝试 | 结果 | 结论 |
|---|---|---|
| fitter 观测年龄窗 | sonar progress 38.2% → 17.0% | 硬删除旧点破坏初始捕获 |
| fitter 时间半衰权重 | 最好仍未 endpoint | 只改变误差均值，不解决闭环 |
| 声呐航向与 PCA 航向门控 | 17.0%–37.9% | 改变状态机节奏但不能建立持续局部轨迹 |
| 声呐在线时持续接管航向 | 38.1%，未 endpoint | 单步航向正确不等于路径状态稳定 |
| TRACK zig-zag / cross-track 参数矩阵 | 最好 61.9% 但距路由 1625 m | route progress 为投影假阳性 |
| 4× 等比放大 | 约 37.8% | 尺寸放大不是解法 |

## 3. 真实根因

当前感知只有 `WeightedSlidingWindowFitter` 的线性 PCA 模型。它把峰值点和声呐点混为无时序状态的点云，输出一条直线。

迷宫场景下失败链路为：

1. SEARCH/ALIGN 阶段少量高 SNR 或高置信点进入 fitter。
2. bootstrap 后 PCA 输出局部直线，但航向可偏离 lane 真值约 17°。
3. TRACK 根据错误纵向基准前进，车辆在 lane1 中段前就漂出声呐量程。
4. 声呐脱窗后没有新的位置观测，fitter 继续冻结旧方向。
5. 控制器继续沿旧方向漂离，后续 route progress 可能只是最近点投影跳变。

这不是物理不可行域：磁场强度、声呐量程、lane 串扰都不构成首要限制。问题是估计器没有“沿时间推进的电缆状态”。

## 4. 目标架构

新增一个局部路径状态估计层，位置位于 perception 包内，职责独立于控制器：

```text
SonarReading / peak observations
        │
        ▼
LocalCableStateEstimator
        ├── 观测门控：时效、距离、航向一致性、离群点
        ├── 局部模型：全局直线 / 局部直线 / 圆弧 / 样条小窗
        ├── 时序状态：当前位置 s、切向 heading、曲率 kappa、横向残差
        └── 健康度：mature / stale / reacquire_required
        │
        ▼
PerceptionState
        ├── fused_heading_deg
        ├── estimated_cable_point_xy_m
        ├── fit_result（兼容旧接口）
        └── deployment_reacquire_required
```

控制器仍只消费 `PerceptionState`，保持 GUI/Logic 与 perception/control 分离。

## 5. 实施阶段

### Phase A：局部直线状态替换点云 PCA

目标：在直线 lane1 上消除 17° 初始偏航与旧点污染。

实现：

- 新增 `perception/local_path.py`。
- 输入声呐点与声呐航向，维护时间窗口内的局部状态。
- 先独立实现全局直线、局部直线与圆弧模型：圆弧可靠时输出 `arc`；圆弧不可靠但曲线正在转向时输出 `local_line`，只用最近小窗并按新近性加权，使切向随 AUV 前进自动转动；直线段仍保持 `line`。
- 航向观测不是只降权置信度，而是参与局部直线切向融合，避免圆弧估计不可靠时退回静态全局 PCA。
- 输出等价 `FitResult`，不改控制器契约。
- 当前已接入旁路诊断字段：`local_path_model_code`、`local_path_heading_deg`、`local_path_confidence`、`local_path_residual_m`、`local_path_radius_m`。这些字段进入 `RunRecord`。
- 加噪独立测试已覆盖 0.45 m 位置噪声和 5° 航向噪声；调优结果采用 `local_line_window=5`、`heading_blend=0.65`，典型局部切向均值误差约 4–5°。
- 主流程接入采用保守门控：默认关闭，仅在无先验且 `local_path_guidance_enabled=True` 的部署场景中允许 `LOCAL_PATH` 参与 `fused_heading_deg`；case1–6 名义场景不受影响。
- `case_maze_sonar` 接入后 route completion 从 38.2% 提升到 59.6%，但仍未达到 endpoint；`case_maze_no_sonar` 仍为 0.3%，说明无声呐下仍缺少纵向切向观测。
- 已证伪两条后续补丁路线：把 local path 直接接入 FSM 信号锁存会退化到 7.7%；在 SEARCH/ALIGN 中强行压低交叉角会退化到 14.8%。下一步应实现显式重捕获状态，而不是继续放宽 local path 门控。
- 纯感知层新增 `LocalPathTrackingState`：`collecting / line_track / curve_track / reacquire`。该状态机只在 `LocalCableStateEstimator` 内部运行，用于 case6/maze 风格大曲线单元测试；当前 orchestrator 实例化时显式 `state_machine_enabled=False`，避免未验证状态直接进入主 workflow。
- 新增单元测试覆盖 R=60m maze U-turn、R=66m case6 风格 S 曲线，以及观测间隔后的 reacquire。测试目标是局部切向连续、无 ±180° 翻转，并显著优于全局 chord 拟合。

验收：

- 独立单元测试先通过短曲线大转角（如 R=38m、120° 转角）与圆弧禁用场景，证明 `local_line` 会随前进改变角度。
- 旁路诊断可在 `case_maze_sonar` 中稳定产出，并与旧 PCA 航向形成可对比序列。
- 主流程接入后，`case_maze_sonar` 至少优于 38.2% 基线且不引入 case1–6 回归；当前实测为 59.6%，但尚未 endpoint。
- 局部状态机并入主 workflow 前，必须先证明在独立测试中稳定通过 case6/maze 大曲线，再设计独立 reacquire workflow；不得直接把 local 状态接入任务 FSM 锁存。
- case1–6 指标不回归。

### Phase B：圆弧/样条小窗模型

目标：通过第一个 U-turn。

实现：

- 在局部窗口内比较直线模型与圆弧模型的 AIC/残差。
- 圆弧模型只在观测跨度和曲率显著性足够时启用。
- 输出当前位置最近点切向，而不是全局平均 PCA 方向。

验收：

- `case_maze_sonar` 通过第一个 U-turn 并进入 lane2。
- 不允许依赖仿真真值路由。

### Phase C：重捕获与无声呐变种

目标：降低声呐脱窗后的不可恢复性。

实现：

- `deployment_reacquire_required=True` 时控制器进入局部重捕获扫线，而非沿冻结 heading 远离。
- 无声呐变种先验上更困难，应先明确其传感器物理可观测性；若仅靠磁强度不能提供纵向切向，需记录为独立不可行边界，而不是强行过验收。

验收：

- `case_maze_sonar` 达到 `stop_reason=endpoint`。
- `case_maze_no_sonar` 若无法通过，必须给出物理可观测性证明和文档化边界。

### Phase D：感知/任务层可观测区域选择

目标：把重捕获从“控制器临时决定搜索形状”上移为“感知/任务层选择下一块可观测区域”。控制器只执行进入区域、横切区域、报告结果，不再根据陈旧 heading 自行生成搜索目标。

#### D.1 设计判断

Phase B/C/E 已证伪的共同问题是：它们都在控制器层改变路径几何，但没有回答“下一块最可能重新获得观测的区域在哪里”。控制器只能看到最后可信锚点、当前位姿和一个重捕获布尔量，因此容易出现两类失败：

- 无界沿切向推进：投影进度可到 100%，但真实终点距离数公里。
- 有界锚点扫线：车辆被困在早期锚点附近，无法推进到后续 U-turn / lane 区域。

Phase D 的核心不是换一种 zig-zag，而是让 perception/mission 根据观测历史生成候选区域，并把“区域选择、区域优先级、区域过期、区域失败后切换”显式建模。

#### D.2 新增数据流

```text
Perception observations
    ├── sonar cable points / headings
    ├── magnetic peaks / peak zones
    ├── local_path state / confidence / stale age
    └── vehicle trajectory around last detection
            │
            ▼
ObservableRegionSelector  (perception/reacquire_region.py)
    ├── 生成候选区域：forward gate / turn-side gate / last-crossing gate / expanding box
    ├── 给候选区域评分：观测新鲜度、切向一致性、磁峰可见性、离当前车体可达性
    ├── 输出 active_region 与失败/切换原因
    └── 禁止使用仿真真值路线
            │
            ▼
PerceptionState
    ├── deployment_reacquire_required
    ├── reacquire_region_center_xy_m
    ├── reacquire_region_heading_deg
    ├── reacquire_region_half_length_m
    ├── reacquire_region_half_width_m
    ├── reacquire_region_confidence
    └── reacquire_region_reason
            │
            ▼
MissionManager
    ├── SEARCH_ZIGZAG
    ├── LOCK_ALIGN
    ├── TRACK_ACTIVE
    ├── REACQUIRE_REGION   (新增，非终态)
    └── EMERGENCY_SURFACE
            │
            ▼
Controller
    ├── 若有 active_region：驶向区域入口并在区域内横切
    └── 若无 active_region：退回当前 Phase A 简单锚点扇区基线
```

#### D.3 区域候选定义

所有候选区域均在“最后可信局部电缆状态”坐标系中定义，包含 anchor、tangent、normal、timestamp、confidence。候选不允许使用真值 route，只允许使用传感器历史和车辆轨迹。

| 候选区域 | 目的 | 生成方式 | 风险 |
|---|---|---|---|
| `forward_gate` | 沿最后可信切向向前找回同一 lane | anchor + `s_forward` * tangent，宽度按声呐量程/zig-zag 半带设置 | 若 heading 已错，会沿错误方向推进 |
| `turn_side_gate` | U-turn 前后寻找转弯侧区域 | 根据最近局部曲率符号、heading 变化方向、车辆相对锚点侧向漂移，选择 normal 左/右侧 | 曲率符号不稳定时可能选错侧 |
| `last_crossing_gate` | 回到最近磁峰/声呐交叉附近重新建立观测 | 使用最近峰区/声呐点云的加权中心和局部 heading | 可能回到旧点，推进不足 |
| `expanding_box` | 以上候选失败后的有界兜底 | 以最后可信锚点为中心按时间扩大，但必须有限半径并有失败切换 | 半径过大时类似无目标搜索 |

首轮实现优先只启用 `forward_gate + turn_side_gate`。`last_crossing_gate` 和 `expanding_box` 作为诊断/兜底，不直接替代主策略。

#### D.4 区域评分

区域选择器输出最高分候选。建议评分项：

```text
score =
  0.30 * recency_score
+ 0.25 * heading_consistency_score
+ 0.20 * magnetic_visibility_score
+ 0.15 * reachability_score
+ 0.10 * exploration_bonus
- stale_penalty
- revisit_penalty
```

- `recency_score`：区域锚点来自越新的声呐/峰值观测，分越高。
- `heading_consistency_score`：候选 heading 与 local_path / sonar heading / magnetic heading 的一致性。
- `magnetic_visibility_score`：候选附近最近一次横切是否出现峰区、SNR、峰宽合理。
- `reachability_score`：当前位姿到区域入口距离和所需转向半径是否合理。
- `exploration_bonus`：尚未尝试过的 turn-side 区域加分，避免困在同一锚点。
- `revisit_penalty`：同一候选区域失败后，在 cooldown 内降权。

#### D.5 状态边界

新增 `MissionState.REACQUIRE_REGION`，只在部署/maze 无强先验场景启用：

- `TRACK_ACTIVE -> REACQUIRE_REGION`：local path stale、声呐脱窗、且 selector 有 `active_region`。
- `REACQUIRE_REGION -> LOCK_ALIGN`：区域内重新获得 sonar 或磁峰，且连续若干帧信号存在。
- `REACQUIRE_REGION -> SEARCH_ZIGZAG`：区域失败、selector 无可用候选、或超过最大尝试时间。
- `REACQUIRE_REGION -> EMERGENCY_SURFACE`：维持原低置信 emergency 逻辑，但阈值不得比当前主线更激进。

任务层只决定“进入哪个任务态”和“是否执行区域重捕获”。它不直接计算航向。控制器只消费 `reacquire_region_*` 字段。

#### D.6 控制器职责收缩

控制器保留三种执行器：

1. `track_executor`：现有 TRACK/curve_track 压线与低幅 zig-zag。
2. `search_executor`：现有 SEARCH/LOCK zig-zag。
3. `region_reacquire_executor`：新增，输入 region center/heading/half_length/half_width：
   - 若车不在区域内，先航向区域入口点。
   - 进入区域后，只在 region 局部坐标中横切，不允许沿 stale heading 无限推进。
   - 区域失败/成功由 mission/perception 判定，控制器不自作切换。

Phase C 已加入的 `reacquire_zigzag_enabled` 保持默认关闭；Phase D 的 region executor 可以复用其局部横切代码，但必须由 `active_region` 驱动。

#### D.7 实施步骤

1. **旁路诊断先行**：新增 `perception/reacquire_region.py` 和 `ObservableRegion` dataclass，只输出诊断字段，不接控制。
2. **记录器扩展**：把 region center、heading、half_width、score、reason 写入 `RunRecord`，用于定位 61.7% 后 selector 是否选到合理区域。
3. **独立单元测试**：用合成 lane + U-turn 点列测试 `forward_gate`、`turn_side_gate`，不跑完整仿真。
4. **影子模式仿真**：`case_maze_sonar` 主控制仍走 Phase A，离线检查 region 是否落在后续可观测走廊附近；此阶段不要求 route 提升。
5. **任务层接入**：给 `MissionInput/MissionDecision` 加 region 摘要，新增 `REACQUIRE_REGION`，仅在 `local_path_guidance_enabled=True and not use_nominal_route_prior` 启用。
6. **控制器接入**：实现 `region_reacquire_executor`，并保留 Phase A 锚点扇区作为无 region 时 fallback。
7. **参数扫描**：只扫描 selector 级参数，例如 forward distance、turn-side offset、region half width、candidate cooldown；不再扫描控制器任意 zig-zag 形状。

#### D.8 验证标准

- 必须同时检查 `endpoint_completed=1`、`stop_reason=endpoint`、最终 endpoint 距离，而不是只看 route progress。
- `case_maze_sonar` 第一目标：超过当前真实基线 61.7%，且最终 endpoint 距离小于当前约 265 m。
- 第二目标：达到 `stop_reason=endpoint`。
- `case6` mean heading error 不得劣化超过 2°。
- `case1`–`case5` 不启用 `REACQUIRE_REGION`，指标必须保持既有矩阵。
- 影子模式必须先证明 selector 输出合理；若 selector 输出不合理，不进入控制器接入。

#### D.9 预期风险

- 若传感器历史没有足够信息区分 U-turn 左/右侧，`turn_side_gate` 会退化为猜测；此时应记录为可观测性不足，而不是继续加控制器补丁。
- 若 `local_path` 的 heading 已严重偏置，`forward_gate` 会把错误 heading 制度化；需要 heading consistency 和 revisit penalty 抑制。
- 无声呐 maze 仍可能缺少纵向切向观测，Phase D 先针对 `case_maze_sonar`，`case_maze_no_sonar` 只做诊断，不设强通过目标。

## 6. 当前执行记录

- 已新增 `local_path_tracking_state_code` 与 `deployment_reacquire_required` 诊断通道，进入 `RunRecord`。
- 主流程采用双估计器边界：legacy local-line 估计器继续负责 `LOCAL_PATH` 引导；状态机估计器只导出 `tracking_state/reacquire` 旁路状态，避免直接改变既有引导基线。
- Phase A 重捕获最小接入：当 local path 超过 `lost_timeout_s` 未刷新时置 `deployment_reacquire_required=True`，控制器围绕最近可信电缆锚点做有界扇区搜索，不再使用随车辆漂移的陈旧投影点作为锚点。
- Phase B 尝试：给局部状态机增加空间观测间隔，目的是让 R=60m 曲线窗口获得足够跨度；实测会破坏 maze 闭环（0%/26%/59.5% 多组均未优于 61.7%），因此当前仅保留配置能力，maze 默认关闭该空间门控。
- Phase E 尝试：将重捕获升级为扩张四象限搜索，并加入最小运行时间门槛；实测 0%/3.4%/8.2%，均明显劣于简单锚点扇区搜索。当前保留 Phase A 的简单有界扇区重捕获作为最佳基线。
- Phase C zig-zag 搜索尝试：新增可配置的锚点局部 zig-zag 重捕获模式。无界沿切向推进会产生 `route=100%` 的投影假阳性（终点真实距离约 3.35 km）；有界局部扫线及小范围参数扫描最高仅 9.6%，会把车辆困在早期锚点附近。TRACK/CURVE 穿越角扫描确认 `track_active_zigzag_angle_deg=10°` 是当前必要基线，0°/20° 均退化；局部重捕获 zig-zag 默认关闭。
- 当前 `case_maze_sonar` 最佳实测 route progress 为 61.7%，`stop_reason=duration`，终点距离约 265 m；重捕获带来小幅真实增益但仍未闭环到 endpoint，说明 Phase B/C/E 的朴素补丁路线已证伪。
- Phase D 初步接入：新增感知层 `ObservableRegionSelector`、`reacquire_region_*` 诊断字段、`REACQUIRE_REGION` 任务态和控制器区域执行器。2× maze 上曾达到 79.5% route progress，但 endpoint 距离仍约 661 m，说明区域选择能推进但仍有方向/区域切换错误。按最新诊断策略，maze 缩回 1× prototype 并停止网格搜参。
- 1× maze 代表点测试：当前参数 `forward=48m, lateral=60m, half_width=24m, half_length=36m` 达到 `route=76.5%`、`stop=duration`、`route_dist=16.1m`、endpoint 距离约 `269.1m`，保住约 70% 推进；保守代表点 `36/40/18/30` 退化到 43.0%，激进代表点 `72/90/36/54` 退化到 44.2%。后续只选代表点，不再做大地图网格搜参。
- Phase D shadow 化：`reacquire_region_enabled` 仅控制 selector 诊断，新增 `reacquire_region_control_enabled` 作为任务接管开关，1× maze 默认关闭控制接管。shadow 结果为 `route=60.2%`、`stop=duration`，没有进入 `REACQUIRE_REGION`，但后半程持续输出 `turn_side_gate`，证明区域信号存在；临时开启控制接管的 current 代表点可到 `route=85.1%`，但 endpoint 距离约 `304.7m`，说明推进增加但终点方向仍不可靠，因此不把控制接管作为默认主线。
- Phase D 通用 FSM 切换逻辑：`TRACK_ACTIVE -> REACQUIRE_REGION` 不再由单帧布尔量触发，而要求 `reacquire_region_control_enabled && reacquire_required && region_available && region_confidence >= min_confidence` 连续满足 `entry_streak`；`REACQUIRE_REGION -> LOCK_ALIGN` 要求信号恢复且 `reacquire_required=False` 连续满足 `recovery_streak`；区域消失先进入 `unavailable_hold`，超过 hold 才回 `SEARCH_ZIGZAG`；单次区域重捕获超过 `max_duration` 也回 `SEARCH_ZIGZAG`。该逻辑可用于 shadow/active 两种模式，避免把 selector 诊断直接绑定到控制接管。
- Phase D 代表点调优：确认 1× active current 未在 2200s endpoint 的主因是时间预算不足，而非区域方向完全错误；`duration=2600s` 后默认 `case_maze_sonar` 达到 `route=99.5%`、`stop=endpoint`，最终 endpoint 距离约 `11.4m`、route 横向距离约 `5.4m`。因此 1× sonar maze 默认打开 `reacquire_region_control_enabled=True`，no-sonar 仍关闭。尝试“稳定 FSM”代表点（entry/recovery 更长、max_duration=120s）退化到 42.4%，仅放宽 `max_duration=75s` 退化到 39.0%，说明当前成功依赖 region/track 频繁交替补观测，不能简单追求低切换数。
- `case6` mean heading error 保持 27.6°；`case1`–`case5` 回归结果与既有矩阵一致，无主线回归。
