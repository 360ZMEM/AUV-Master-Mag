# 行为树方法论

## 1. 从状态机到行为树

### 1.1 传统状态机的局限

传统的有限状态机（FSM）在复杂决策场景下面临挑战：

```python
# 传统 FSM 的嵌套逻辑
if confidence > 0.7:
    if peak_detected:
        if not safe_lock:
            if sonar_online:
                # ... 嵌套过深
            else:
                # ...
        else:
            # ...
    else:
        # ...
else:
    if has_detection_history:
        # ...
    else:
        # ...
```

**问题**：
1. 条件嵌套过深，难以理解
2. 新增行为需要修改多处代码
3. 优先级关系不明确
4. 难以测试和调试

### 1.2 行为树的优势

行为树（Behavior Tree）采用**优先级评估**模式：

```
每个时刻：
  for node in nodes (按优先级排序):
    if node.should_run(context):
      return node.run(context)
```

**优势**：
1. **清晰的优先级**：高优先级行为先评估
2. **模块化**：每个行为节点独立
3. **易扩展**：新增行为只需添加节点
4. **易测试**：每个节点可独立验证

## 2. 行为节点的六个层次

### 2.1 优先级顺序（从高到低）

```
┌─────────────────────────────────────────────────────────┐
│ 1. SafeLockNode      (安全锁)                           │
│    - 条件：激活 + 有检测 + 在丢失窗口内                  │
│    - 行为：强制贴缆，宽度=0                              │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│ 2. TurnNode           (过峰转向)                         │
│    - 条件：检测到峰值 + 有融合航向                        │
│    - 行为：翻转 Zig-zag 方向，保持宽度                    │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│ 3. HoldNode           (高置信度保持)                     │
│    - 条件：高置信度 + 非弱信号                           │
│    - 行为：贴缆航行，宽度=0.55×正常                      │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│ 4. ApproachNode       (中等置信度接近)                   │
│    - 条件：有融合航向 + 中等置信度                       │
│    - 行为：正常 Zig-zag 跟踪                             │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│ 5. LostNode           (丢失恢复)                         │
│    - 条件：有历史 + 超时 + 声纳离线 + 无盲航向            │
│    - 行为：搜索速度，可能进入螺旋搜索                    │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│ 6. SearchNode         (默认搜索)                         │
│    - 条件：总是返回 true（兜底）                         │
│    - 行为：搜索速度，正常 Zig-zag                        │
└─────────────────────────────────────────────────────────┘
```

### 2.2 每个节点的详细逻辑

#### SafeLockNode（安全锁）

**目的**：防止跟踪陈旧信息

**触发条件**：
```python
active and has_detection and age <= lost_timeout
```

**物理意义**：
- 刚刚检测到峰值，但信号突然崩溃
- 可能已经越过电缆，需要"锁住"当前方向
- 避免在盲区被误导

**输出决策**：
```python
BehaviorDecision(
    mode=TURN,
    base_heading=fused_heading,
    speed=0.9 × cruise_speed,
    zigzag_width=0.0,        # 强制贴缆
    force_centerline=True    # 不执行 Zig-zag
)
```

#### TurnNode（过峰转向）

**目的**：响应峰值检测，翻转 Zig-zag

**触发条件**：
```python
peak_detected and fused_heading is not None
```

**物理意义**：
- 刚刚完成一次穿缆
- 需要翻转方向准备下一次横切
- 是 Zig-zag 的核心触发器

**输出决策**：
```python
BehaviorDecision(
    mode=TURN,
    base_heading=fused_heading,
    speed=cruise_speed,
    zigzag_width=normal_width,  # 保持当前宽度
    guidance_source="MAGNETIC_PEAK"
)
```

#### HoldNode（高置信度保持）

**目的**：高置信度下的紧密跟踪

**触发条件**：
```python
(fused_heading is not None and confidence >= high_threshold and not weak_signal)
or (memory_hold_ready)
```

其中 `memory_hold_ready` 的条件：
- 有融合航向
- 引航源是 MEMORY/BLIND/BLIND_RECOVERY
- 检测年龄 ≥ guidance_memory_timeout
- 置信度 ≥ low_threshold
- 拟合残差 ≤ 3.5m（部署模式 1.8m）

**物理意义**：
- 对电缆位置和方向有高信心
- 可以缩小搜索宽度，提高精度
- 在标准模式下更宽松，部署模式下更严格

**输出决策**：
```python
BehaviorDecision(
    mode=HOLD,
    base_heading=fused_heading,
    speed=cruise_speed,
    zigzag_width=0.55 × normal_width,  # 缩小宽度
    guidance_source="FUSION_HOLD"
)
```

#### ApproachNode（中等置信度接近）

**目的**：正常跟踪模式

**触发条件**：
```python
fused_heading is not None and confidence >= low_threshold
```

**物理意义**：
- 有一定的跟踪置信度
- 但还不足以进入 HOLD 模式
- 是最常用的跟踪模式

**特殊处理**：如果声纳在线且信号弱，优先使用声纳

**输出决策**：
```python
source = "SONAR" if (sonar_online and weak_signal) else "MAGNETIC"
BehaviorDecision(
    mode=APPROACH,
    base_heading=fused_heading,
    speed=0.95 × cruise_speed,  # 略微减速
    zigzag_width=normal_width,
    guidance_source=source,
    force_centerline=(source == "SONAR")  # 声纳强制贴缆
)
```

#### LostNode（丢失恢复）

**目的**：处理信号丢失

**触发条件**：
```python
has_detection_history and age > lost_timeout and sonar_offline and blind_heading is None and fused_heading is None
```

**物理意义**：
- 曾经检测到过电缆
- 但现在已经长时间丢失
- 没有声纳辅助，没有盲航向，没有融合航向

**部署模式的特殊处理**：
- 如果需要重新获取 → 螺旋搜索
- 如果没有融合航向且没有盲航向 → 螺旋搜索

**输出决策**：
```python
BehaviorDecision(
    mode=LOST (或 SPIRAL_SEARCH),
    base_heading=blind_heading or nominal_heading,
    speed=search_speed,
    zigzag_width=normal_width,
    guidance_source="LOST_RECOVERY" (或 "REACQUIRE_SPIRAL")
)
```

#### SearchNode（默认搜索）

**目的**：兜底行为，总是有输出

**触发条件**：
```python
True  # 总是返回 true
```

**部署模式的特殊处理**：
1. 无检测历史 → 螺旋搜索
2. 置信度不足 → 螺旋搜索
3. 需要重新获取 → 螺旋搜索
4. 无航向估计 → 螺旋搜索

**如果有盲航向**：
- 使用盲航向进入 APPROACH 模式
- Zig-zag 宽度缩小到 35%

**输出决策**：
```python
heading = intercept_heading (if distance > 3m) else nominal_heading
if blind_heading is not None:
    heading = blind_heading

BehaviorDecision(
    mode=SEARCH (或 APPROACH 或 SPIRAL_SEARCH),
    base_heading=heading,
    speed=search_speed,
    zigzag_width=normal_width (或 0.35×normal_width 或 0),
    guidance_source="BLIND" or "NOMINAL" or "BOOTSTRAP_SPIRAL"...
)
```

## 3. 行为上下文（BehaviorContext）

### 3.1 上下文的作用

行为树不是静态的，每个时刻都需要评估当前的"情境"。这个情境由 30+ 个参数组成。

### 3.2 关键上下文参数

| 类别 | 参数 | 物理意义 |
|------|------|----------|
| **时间** | time_s | 当前仿真时间 |
| **导航** | nominal_heading_deg | 名义路由方向 |
| **导航** | intercept_heading_deg | 拦截名义路由的方向 |
| **导航** | nominal_distance_m | 到名义路由的距离 |
| **感知** | confidence | 当前融合置信度 |
| **感知** | has_detection_history | 是否曾经检测到过 |
| **感知** | last_detection_age_s | 距离上次检测的时间 |
| **感知** | fused_heading_deg | 融合后的电缆航向 |
| **感知** | blind_heading_deg | 盲从航向（无拟合时） |
| **感知** | sonar_status | 声纳状态（OFFLINE/DEGRADED/ONLINE） |
| **感知** | weak_signal_flag | 信号是否过弱 |
| **感知** | safe_lock_active | 安全锁是否激活 |
| **感知** | peak_detected | 当前时刻是否检测到峰值 |
| **控制** | zigzag_width_m | 当前 Zig-zag 宽度 |
| **控制** | fit_residual_m | 拟合残差 |
| **部署** | deployment_mode | 是否部署模式 |
| **部署** | deployment_heading_confidence | 部署模式航向置信度 |
| **部署** | deployment_reacquire_required | 是否需要重新获取 |

### 3.3 门限参数

| 参数 | 典型值 | 作用 |
|------|--------|------|
| high_confidence_threshold | 0.7 | HOLD 模式阈值 |
| low_confidence_threshold | 0.3 | APPROACH 模式阈值 |
| lost_timeout_s | 4.0 | 信号丢失超时 |
| guidance_memory_timeout_s | 3.0 | 记忆导航超时 |
| consecutive_miss_threshold | 4 | 连续未命中阈值 |
| spiral_entry_window_s | 8.0 | 螺旋搜索进入窗口 |

## 4. 行为决策（BehaviorDecision）

### 4.1 决策输出结构

```python
@dataclass
class BehaviorDecision:
    mode: BehaviorMode              # 行为模式
    base_heading_deg: float          # 基础航向（未加 Zig-zag）
    speed_mps: float                 # 速度
    zigzag_width_m: float            # Zig-zag 宽度
    guidance_source: str             # 导航源标识
    force_centerline: bool = False   # 是否强制贴缆（不加 Zig-zag）
```

### 4.2 从决策到控制指令

行为树的输出不是最终的控制指令，需要经过后处理：

```
BehaviorDecision
    ↓
应用 Zig-zag 偏置
    ↓
desired_heading = base_heading + leg_sign × crossing_angle
    ↓
限制偏航速率
    ↓
最终 GuidanceCommand
```

**Zig-zag 偏置计算**：

```python
crossing_angle = arctan(zigzag_width / lookahead_distance)
crossing_angle = clip(crossing_angle, min_angle, max_angle)
```

典型值：
- lookahead_distance = 10m
- zigzag_width = 2.5-9m
- crossing_angle = 14-42°

## 5. 特殊机制

### 5.1 转弯恢复（Turn Recovery）

**目的**：防止在转弯后连续丢失

**机制**：
1. 记录 TURN 模式的时刻
2. 在之后的 8 秒窗口内，连续计数未命中
3. 如果连续未命中 ≥ 4 次，强制进入螺旋搜索

**代码**：
```python
def _should_force_spiral_search(context):
    in_recovery_window = (time - last_turn_time) <= spiral_entry_window_s
    return (turn_recovery_active and in_recovery_window
            and sonar_offline and consecutive_misses >= threshold)
```

### 5.2 安全锁（Safe Lock）

**触发条件**（感知层）：
1. **准则 A**：信号崩溃 + 过度位移
   - 当前强度 < 0.6 × 上次峰值强度
   - 距离上次峰值 > 1.5 × 理想场宽

2. **准则 B**：梯度方向不一致
   - 梯度方向与拟合线法线的夹角 > 60°

**效果**：
- 拟合结果失效
- 强制贴缆（宽度=0）
- 持续 2 秒

### 5.3 螺旋搜索（Spiral Search）

**适用场景**：
- 部署模式下无先验
- 信号丢失后无法恢复
- 需要重新获取电缆

**螺旋参数**：
- 初始半径 = 最小转弯半径
- 半径增长率 = 0.5 m/s
- 最大半径 = 60m

**偏航速率**：
```python
yaw_rate = (speed / turning_radius) × leg_sign
```

**终止条件**：
- 检测到峰值
- 达到最大半径

## 6. 部署模式的特殊处理

### 6.1 与标准模式的差异

| 特性 | 标准模式 | 部署模式 |
|------|----------|----------|
| 名义路由先验 | 启用 | 禁用 |
| 初始行为 | 拦截路由 | 螺旋搜索 |
| Hold 条件 | 较宽松 | 更严格 |
| 拟合接受 | 容忍大残差 | 要求小残差 |
| 航向估计 | 多假设消歧 | 梯度增强 |

### 6.2 部署模式下的螺旋搜索类型

| 类型 | 触发条件 | 物理意义 |
|------|----------|----------|
| BOOTSTRAP_SPIRAL | 无检测历史 | 从零开始搜索 |
| DEPLOYMENT_SPIRAL | 置信度不足 | 增强置信度 |
| REACQUIRE_SPIRAL | 需要重新获取 | 恢复丢失跟踪 |
| RECOVERY_SPIRAL | 无航向估计 | 建立方向估计 |

```
                    ┌──────────────┐
                    │   SEARCH     │
                    │  (初始/搜索)  │
                    └──────┬───────┘
                           │ 检测到峰值
                    ┌──────▼───────┐
                    │    TURN      │
                    │  (翻转方向)   │
                    └──────┬───────┘
                           │ 持续检测
                    ┌──────▼───────┐
                    │  APPROACH    │
                    │  (中等置信)   │
                    └──────┬───────┘
                           │ 置信度升高
                    ┌──────▼───────┐
                    │    HOLD      │
                    │  (高置信)    │
                    └──────┬───────┘
                           │ 信号丢失
                    ┌──────▼───────┐
                    │    LOST      │
                    │  (丢失恢复)   │
                    └──────┬───────┘
                           │ 超时/无航向
                    ┌──────▼───────┐

                    └───────────────┘

特殊情况：
- 任何时刻 ──峰值崩溃 + 安全锁──► SAFE_LOCK (贴缆)
- SPIRAL ──检测到峰值──────► TURN/SEARCH
```

## 8. 性能权衡

### 8.1 置信度门限的影响

| high_threshold | low_threshold | 效果 |
|----------------|---------------|------|
| 0.9 | 0.5 | 保守：难进入 HOLD，更宽松的搜索 |
| 0.7 | 0.3 | 平衡：默认值 |
| 0.5 | 0.2 | 激进：易进入 HOLD，更快贴缆 |

### 8.2 丢失超时的影响

| lost_timeout_s | 效果 |
|----------------|------|
| 2.0 | 快速声明丢失，更频繁搜索 |
| 4.0 | 平衡：默认值 |
| 8.0 | 慢速声明丢失，更长记忆 |

### 8.3 Zig-zag 宽度的影响

| zigzag_width | 效果 |
|--------------|------|
| 2-3m | 精确跟踪，易丢失 |
| 4-6m | 平衡：默认值 |
| 8-9m | 宽松搜索，慢响应 |

## 9. 调试与监控

### 9.1 关键监控指标

| 指标 | 正常范围 | 异常指示 |
|------|----------|----------|
| mode | SEARCH/APPROACH/HOLD | 长时间 SEARCH/L |
| guidance_source | MAGNETIC/SONAR/MEMORY | 长时间 SEARCH |
| confidence | 0.3-1.0 | < 0.3 表示不可靠 |
| last_detection_age_s | 0-4s | > 4s 表示丢失 |
| fit_residual_m | 0-5m | > 8m 表示拟合差 |
| zigzag_width_m | 2-9m | 异常宽度 |

### 9.2 模式转换频率

**正常**：
- SEARCH → APPROACH → HOLD（逐渐收敛）
- 定期 TURN（每次穿缆）

**异常**：
- 频繁 SEARCH ↔ LOST 振荡
- 长时间停留在 SPIRAL_SEARCH
- 无法进入 HOLD 模式

### 9.3 典型场景序列

**case1（标准直线）**：
```
SEARCH(0-5s) → TURN → APPROACH → HOLD(持续)
```

**case2（转弯）**：
```
HOLD → TURN → 残差增大 → APPROACH → HOLD(新方向)
```

**case3（大噪声）**：
```
SEARCH → APPROACH ↔ LOST → SEARCH → ...
```

## 10. 总结

行为树方法论的三大支柱：

1. **优先级清晰**：高优先级行为总是先评估
2. **模块独立**：每个行为节点可独立测试和修改
3. **上下文驱动**：决策基于完整情境而非单一条件

**核心公式**：

$$
\text{行为} = \text{优先级排序} \times \text{情境评估} \times \text{模块化执行}
$$

与硬编码的状态机相比，行为树提供了一种更灵活、更可扩展、更易理解的决策架构。
