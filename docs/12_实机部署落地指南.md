# 实机部署落地指南

> **定位**：把仿真闭环迁移到真实 AUV 的工程落地清单——传感器必要性评估、wrapper 落地建议、
> API 可重构点、海缆路由整合、实测噪声模型注入。
> **配套**：扩展规范见 [`11_二次开发指南.md`](11_二次开发指南.md)；现有实验后端抽象在
> [`experimental/simulator_connector.py`](../src/auv_mag_tracking/experimental/simulator_connector.py)。
> **核心原则**：在系统边界做薄 wrapper，**不污染**核心感知/控制/任务逻辑——它们只认数据契约，不认数据来源。

## 2. 仿真闭环 vs 实机闭环

当前 [`main_viz.py`](../src/auv_mag_tracking/main_viz.py) 每步（仿真）：

```
合成信号电流 → 合成电缆磁场(环境真值) → magnetometer.sample_block ┐
synth attitude → imu.observe                                      │
cable_truth(真值) → sonar.sample / burial_observer.observe        ├→ perception.update → PerceptionState
                                                                  │
                          ↓                                       ┘
              controller.update → GuidanceCommand → propagate_vehicle(仿真运动学) → pose
```

实机需替换的环节（其余原样保留）：

| 环节 | 仿真 | 实机替换 |
|---|---|---|
| 磁场来源 | 环境真值 + 传感器噪声模型 | **真实磁力计驱动** |
| 姿态来源 | `apply_attitude_profile` + `IMUSimulator` | **真实 IMU/AHRS** |
| 声呐来源 | `SonarModel`（吃真值） | **真实声呐驱动** |
| 车辆位置 | 由 `propagate_vehicle` 积分 pose | **真实导航解算（INS/DVL/USBL）** |
| 运动学出口 | `propagate_vehicle` 内部推进 | **`GuidanceCommand` 发布给 AUV 自驾仪** |
| 埋深真值 | `cable_truth.burial_depth_m` | **不可得**（仅用于仿真评估，实机置 NaN） |

> 关键认知：仿真把"导航位置"藏在 `pose` 里。实机必须显式引入**导航解算**作为一路传感器，
> 否则 peak 位置/中心线拟合/横偏都没有世界坐标可落。

## 3. 传感器必要性评估

| 传感器 | 必要性 | 理由 | 实机最低要求 |
|---|---|---|---|
| **三轴矢量磁力计** | **必需（主传感器）** | 全链路的信号根源；AC 穿缆 → 带通/RMS → 峰值/中心线/埋深反演 | 采样率 ≥ 4×信号频率（50 Hz→≥200 Hz，推荐 ≥500 Hz）；矢量三轴；量程覆盖背景场 ±100 µT |
| **IMU / AHRS** | **必需** | Sensor→Body→NED 变换链的前提；无姿态无法把磁场旋到 NED 扣背景、提 AC 方向（case4 专验此链路在 ±18° roll 下稳定） | 航向 + 俯仰 + 横滚；航向噪声 ≲ 1°，倾角噪声 ≲ 0.5° |
| **导航解算（INS/DVL/USBL）** | **必需（仿真隐含项）** | perception 消费 `vehicle_position_xy_m`；峰位、PCA 拟合、横偏均在世界 XY | 提供连续 NED 位置 + 速度；漂移受控（DVL 辅助） |
| **前视/侧扫声呐** | **强烈推荐（可降级）** | 给 PCA 中心线拟合提供**无歧义方向**，化解磁穿缆的 ±90° 歧义；系统磁导航主导（71–99.9%），声呐缺失会退化到名义路由先验/盲航向 | 输出电缆相对方位/距离；可 `degraded` 工作 |
| **埋深传感器** | **可选** | 磁法埋深反演（标定幅度法）已能无专用传感器给埋深（case1 MAE 0.056 m）；直接埋深通道可缺省 | 若有则填 `BurialDepthMeasurement`；否则恒 `valid=False` |

> 一句话：**磁力计 + IMU + 导航** 是硬必需三件套；**声呐**强烈建议（决定方向鲁棒性）；**埋深传感器**可不要（磁法反演替代）。

## 4. Wrapper 落地建议

### 4.1 适配方向：`RawSensorBundle` → perception 的四个输入契约

`perception.update(...)` 的真实签名（见 [`perception/orchestrator.py`](../src/auv_mag_tracking/perception/orchestrator.py#L306)）只认这些数据类，**与来源无关**：

```python
perception.update(
    reading: MagnetometerReading,        # sensor_model.py
    pose_measurement: PoseMeasurement,   # sensor_model.py
    vehicle_position_xy_m: np.ndarray,
    burial_measurement: BurialDepthMeasurement,
    true_burial_depth_m: Optional[float] = None,  # 仅评估用；实机不提供
    sonar_reading: Optional[SonarReading] = None,
    signal_features: Optional[ProcessedSignalFeatures] = None,
)
```

wrapper 的职责就是把 `RawSensorBundle`（实机原始数据）翻译成上面这些：

| RawSensorBundle 字段 | 目标契约 | wrapper 需要做的转换 |
|---|---|---|
| `magnetometer_block_nt` (N×3) | `MagnetometerReading` | 包装为块（`sample_block_sensor_nt`/`sample_times_s`）；`cable_strength_nt`/`weak_signal_flag` **改由 DSP 输出派生**（见 §5-1，不能再靠真值） |
| `imu_heading_deg`（+俯仰/横滚） | `PoseMeasurement` | 直接填；补 `speed_mps`（来自导航） |
| `sonar_relative_position_body_m` + `sonar_heading_deg` | `SonarReading` | 用导航 pose 把 body 相对位/航向换算成 `estimated_position_ned_m`/`estimated_heading_ned_deg`；`confidence` 由真实声呐质量派生 |
| `burial_depth_m`（可选） | `BurialDepthMeasurement` | 有则 `valid=True`，无则 `valid=False` |
| `vehicle_position_ned_m` | `vehicle_position_xy_m` | 来自 INS/DVL/USBL 的 NED 位置 |

### 4.2 已落地的薄 wrapper 形态

已在 [`experimental/deployment_sensor_source.py`](../src/auv_mag_tracking/experimental/deployment_sensor_source.py) 新增 `DeploymentSensorSource`（不进 `experimental/__init__.py` 自动导出，避免默认仿真路径加载额外依赖）：

```python
class DeploymentSensorSource:
    """把 HoloOceanConnector 的 RawSensorBundle 适配成 perception 的输入契约。"""
    def __init__(self, connector: HoloOceanConnector, scenario: ScenarioConfig): ...
    def poll(self, time_s: float, pose: Pose) -> SensorFrame:
        bundle = self.connector.recv_sensor_updates()   # → RawSensorBundle
        ...  # 纯翻译，无算法
```

主循环通过 `AuvCableTrackingSimulation(..., sensor_source=...)` 在「合成传感器」与「`DeploymentSensorSource`」之间二选一——核心 `perception/controller/mission` 不感知数据来源。

### 4.3 运动学出口 wrapper

实机不再直接调用 `propagate_vehicle`，而是实现一个 `VehicleBackend.advance(...)`：把 `GuidanceCommand`
（`desired_heading_deg`/`speed_mps`/`yaw_rate_deg_s`）经 `connector.send_command(cmd)` 送给 AUV 自驾仪，下一帧 pose 从导航读回。
默认仿真仍使用 `KinematicVehicleBackend`，行为与原来一致。

## 5. API 重构状态

第一轮已完成项：

1. **磁强度判定去真值化**：`perception/orchestrator.py` 不再用 `reading.cable_strength_nt` 参与弱信号判定；部署路径以 `ProcessedSignalFeatures.weak_signal_flag` 为准。
2. **`true_burial_depth_m` 降级为评估专用**：`perception.update(..., true_burial_depth_m: Optional[float] = None)`；实机可不传，显示侧输出 `N/A`。
3. **`recv_sensor_updates` 返回类型统一**：ABC / Mock / Null 均返回 `RawSensorBundle`。
4. **传感器源抽象**：`AuvCableTrackingSimulation` 支持注入 `SensorSource`，默认仍走原仿真传感器链。
5. **命令出口抽象**：`AuvCableTrackingSimulation` 支持注入 `VehicleBackend`，默认仍走 `KinematicVehicleBackend`。

剩余实机项：
- 实现真实磁力计/声呐/IMU/导航驱动，把其输出填入 `RawSensorBundle`。
- 实现真实 `VehicleBackend`，完成 `GuidanceCommand` 到 AUV 自驾仪命令格式的映射。
- 海试前用录制的 `RawSensorBundle` 做 HIL 回放，确认时序、坐标系和置信度口径无误。

> 这 5 点都不动算法，只在 `main_viz.py` 装配层和 `experimental/` 边界做接口化——符合"薄 wrapper、核心不变"。

## 6. 海缆路由整合（预先先验）

实机通常有**已知的敷缆路由图**（敷设记录 / KP 海图 / 路由调查）。系统已预留接入点：

- 数据结构：[`routes/prior_waypoints.py`](../src/auv_mag_tracking/routes/prior_waypoints.py) `PriorWaypointsRoute`
  ——3–5 个 waypoints + `tolerance_band_m`（默认 ±30 m 走廊），提供 `reference()/heading_deg()/within_tolerance()`。纯数据，无算法耦合。
- 用途：作为名义航向先验与横向走廊约束，给 SEARCH 段方向、给 TRACK 段越界保护。

落地步骤：
1. **坐标转换（整合边界）**：把路由的地理坐标（lat/lon/KP）转到部署用的**局部 NED 切平面**（与导航同基准）。这是唯一需要新增的几何转换。
2. 用转换后的 XY 构造 `PriorWaypointsRoute(waypoints_xy_m=..., tolerance_band_m=路由不确定度)`。
3. 走廊宽度取路由图标称精度 + 导航不确定度，作为 `tolerance_band_m`。
4. **先验强度**：用部署模式（`use_nominal_route_prior=False` + 稀疏 `PriorWaypointsRoute`）而非旧的密集名义路由——
   先验只兜底方向，不喧宾夺主（避免先验误差被当成观测）。

> 注意：路由先验是"辅助方向 + 越界保护"，**不是**真值喂入拟合器（守 C1 精神：先验不进 PCA 中心线）。

## 7. 实测噪声模型注入（闭环 sim-to-real）

系统的 `HighFidelityMagnetometer` 已建模白噪声 + 粉红噪声 + 脉冲 + 量化 + 平台静态干扰
（[`config/__init__.py`](../src/auv_mag_tracking/config/__init__.py) `HighFidelityMagnetometerConfig`）。
实测后用真实数据**标定这些参数**，让仿真回归真实平台，再重跑回归套件复核阈值。

### 7.1 磁力计噪声标定流程

1. **采背景段**（电缆断电 / 远离电缆）：得到背景场 + 平台干扰 + 传感器噪声。
2. 估计：
   - `white_noise_std_nt`：高频段 PSD 平底；
   - `pink_noise_std_nt` / `pink_noise_exponent`：低频 PSD 斜率（1/f^α 拟合）；
   - `impulse_probability` / `impulse_amplitude_nt`：离群尖峰统计；
   - `auv_static_interference_body_nt`：机体系固定偏置（硬铁/软铁；建议做罗盘式标定）。
3. **采有缆段**（电缆通电）：标定埋深反演系数 `K`（`slant = K·I_rms/B_track`，见 [`08_磁法埋深反演.md`](08_磁法埋深反演.md)）。
4. 把上述写入对应场景的 `high_fidelity` 配置块（一个"实测平台"case）。

### 7.2 声呐 / 埋深噪声标定

- 声呐：从真实声呐日志拟合 `prob_detection`、`position_noise_std_m`、`heading_noise_deg`、`buried_loss_factor`
  （[`SonarConfig`](../src/auv_mag_tracking/config/__init__.py)）。
- 埋深通道（若有）：拟合 `burial_depth_noise_std_m`、`burial_depth_dropout_probability`（`SurveyConfig`）。

### 7.3 注入后必须复核

噪声参数变了，阈值不一定还成立。注入后按 [`11`](11_二次开发指南.md) §7 三连复跑：

```bash
python -m pytest tests/ -q
python tools/visualize.py --all
python tools/visualize.py --progress
```

确认 15° 航向硬约束、埋深 MAE、FSM 切换稳定性在"实测噪声"下仍达标——达标后该配置即"部署前数字基线"。
