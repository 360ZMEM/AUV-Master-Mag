# 场景配置指南

## 1. 配置系统架构

### 1.1 配置对象层次

```
ScenarioConfig
├── SignalConfig (信号配置)
├── SensorConfig (传感器配置)
│   └── HighFidelityMagnetometerConfig (高保真磁力计)
├── SonarConfig (声纳配置)
├── EnvironmentConfig (环境配置)
├── VehicleConfig (AUV 配置)
├── TrackingConfig (跟踪配置)
└── SignalProcessingConfig (信号处理配置)
```

### 1.2 配置来源

1. **代码默认值**：`config/__init__.py` 中的 dataclass 默认值
2. **场景预设**：`build_default_scenarios()` 中的显式覆盖
3. **命令行参数**：运行时动态修改

## 2. 核心配置类别

### 2.1 信号配置 (SignalConfig)

```python
@dataclass
class SignalConfig:
    mode: str = "ac_50hz"              # dc, ac_50hz, ac_demo, ac_60hz
    frequency_hz: float = 50.0         # AC 频率
    dc_current_a: float = 0.0          # DC 电流幅值
    ac_current_amplitude_a: float = 600.0  # AC 电流幅值
    phase_rad: float = 0.0             # 相位
    bandpass_half_width_hz: float = 8.0 # 带通半宽
```

**模式说明**：
- `dc`：稳恒电流，直接幅值跟踪
- `ac_50hz`：50Hz 工频（中国/欧洲标准）
- `ac_demo`：10-20Hz 实验频率
- `ac_60hz`：60Hz 工频（北美标准）

**关键权衡**：
| 频率 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| DC | 处理简单 | 无载波、易受干扰 | 稳恒电流测试 |
| 10-20Hz | 低采样率可行 | 非标准、环境干扰多 | 实验对照 |
| 50Hz | 标准工频 | 需要较高采样率 | 实际部署 |
| 60Hz | 北美标准 | 需要较高采样率 | 北美应用 |

### 2.2 传感器配置 (SensorConfig)

```python
@dataclass
class SensorConfig:
    magnetometer_sample_rate_hz: float = 200.0  # 采样率
    noise_std_nt: float = 0.05                  # 噪声标准差
    bias_drift_std_nt_per_s: float = 0.01       # 零偏漂移
    nonorthogonality_deg: float = 0.2           # 非正交误差
    static_rotation_euler_deg: Tuple[float,float,float] = (0,0,0)
    dynamic_range_nt: float = 100000.0          # 动态范围
    imu_heading_noise_deg: float = 0.1          # IMU 航向噪声
    imu_tilt_noise_deg: float = 0.05            # IMU 倾斜噪声
    weak_signal_threshold_nt: float = 18.0      # 弱信号阈值
    high_fidelity: HighFidelityMagnetometerConfig = ...
```

**采样率选择**：
- **100 Hz**：最低要求，50Hz 信号勉强可处理
- **200 Hz**：默认值，每周期 4 个样本
- **1000 Hz**：高保真模式，更好的带通性能

**噪声水平**：
- **0.05 nT**：工业级磁力计（标准模式）
- **150 nT**：手机级磁力计（case3）
- **0.03 nT + 粉红噪声**：高保真模拟

**弱信号阈值**：
- 用于判断信号是否足够强
- 典型值：15-20 nT
- 影响置信度和导航源选择

### 2.3 高保真磁力计配置 (HighFidelityMagnetometerConfig)

```python
@dataclass
class HighFidelityMagnetometerConfig:
    enabled: bool = False                      # 是否启用
    sampling_rate_hz: float = 1000.0           # 采样率
    bit_depth: int = 24                        # 比特深度
    full_scale_nt: float = 100000.0            # 量程
    auv_static_interference_body_nt: Tuple[float,float,float] = (35,-20,15)
    white_noise_std_nt: float = 0.03           # 白噪声
    pink_noise_std_nt: float = 0.08            # 粉红噪声
    pink_noise_exponent: float = 1.0           # 粉红噪声指数
    impulse_probability: float = 0.002         # 脉冲概率
    impulse_amplitude_nt: float = 60.0         # 脉冲幅度
    impulse_decay_samples: int = 8             # 脉冲衰减
```

**何时启用**：
- 测试手机级传感器性能（`case_hf_phone`）
- 测试工业级传感器性能（`case_hf_industrial`）
- 验证算法对真实传感器特性的鲁棒性

**关键参数影响**：
| 参数 | 增大影响 | 典型值 |
|------|----------|--------|
| pink_noise_std_nt | 低频漂移更明显 | 0.08 nT |
| impulse_probability | 更多假峰值 | 0.002 (0.2%) |
| impulse_amplitude_nt | 假峰值幅度更大 | 60 nT |

### 2.4 声纳配置 (SonarConfig)

```python
@dataclass
class SonarConfig:
    mode: str = "degraded"                    # off, degraded, reliable_absence
    max_range_m: float = 15.0                 # 最大范围
    horizontal_fov_deg: float = 120.0          # 水平视场角
    prob_detection: float = 0.70               # 探测概率
    position_noise_std_m: float = 0.45         # 位置噪声
    heading_noise_deg: float = 5.0             # 航向噪声
    buried_loss_factor: float = 0.08           # 埋深损失系数
    update_rate_hz: float = 8.0                # 更新频率
```

**模式说明**：
- `off`：声纳不可用
- `degraded`：声纳降级（低探测概率、高噪声）
- `reliable_absence`：可靠地报告"无电缆"

**使用场景**：
- **off**：纯磁力计验证
- `degraded`：默认模式，部分辅助
- `reliable_absence`：部署模式下的排除性搜索

### 2.5 环境配置 (EnvironmentConfig)

```python
@dataclass
class EnvironmentConfig:
    cable_waypoints_xy_m: Tuple[Tuple[float,float], ...] = ((-320,0),(360,0))  # 电缆控制点
    cable_route_mode: str = "spline"                  # spline, sine, linear
    spline_tension: float = 0.0                        # 样条张力
    sine_amplitudes_m: Tuple[float, ...] = (6.0, 2.5)  # 正弦振幅序列
    sine_wavelengths_m: Tuple[float, ...] = (140.0, 55.0)  # 正弦波长序列
    seabed_depth_m: float = 30.0                       # 海床深度
    seabed_undulation_m: float = 0.8                   # 海床起伏幅度
    burial_depth_m: float = 1.5                        # 埋深
    background_field_ned_nt: Tuple[float,float,float] = (25000,0,42000)
    nominal_route_heading_deg: float = 0.0             # 名义航向
    field_segment_length_m: float = 4.0                # 场离散化长度
    min_cable_curvature_radius_m: float = 50.0         # 允许的最小曲率半径
    validate_curvature_on_build: bool = True           # 构建时检查曲率约束
```

**电缆路由模式**：
- `linear`：直线连接航路点
- `spline`：样条平滑（case2/case6/case8 转弯场景）
- `sine`：在直线基准上叠加多频正弦（`sine_amplitudes_m` × `sine_wavelengths_m`，用于 case5 缓弯）

**曲率约束**（硬约束相关）：
- `min_cable_curvature_radius_m`：构建场景时校验路由最小曲率半径不低于该值（环境设计硬约束 ≥ 25 m）。
- `validate_curvature_on_build=True`：构建期若违反则告警/报错，防止生成不可跟踪的几何。
- 调正弦缓弯时，**波长越长 → 曲率越缓 → minR 越大**（case5 即用此关系把 minR 从 22.4 m 提到 31.1 m）。

**埋深设置**：
- 影响：磁场衰减、声纳探测
- 典型值：1-2 m
- 更深 → 磁信号弱、声纳难探测

### 2.6 AUV 配置 (VehicleConfig)

```python
@dataclass
class VehicleConfig:
    search_speed_mps: float = 1.5          # 搜索速度
    cruise_speed_mps: float = 2.0          # 巡航速度
    min_turning_radius_m: float = 5.0       # 最小转弯半径
    max_yaw_rate_deg_s: float = 30.0        # 最大偏航速率
    altitude_above_seabed_m: float = 3.0    # 离底高度
    pitch_amplitude_deg: float = 2.0        # 俯仰扰动幅度
    pitch_frequency_hz: float = 0.15        # 俯仰扰动频率
    roll_amplitude_deg: float = 5.0         # 滚转扰动幅度
    roll_frequency_hz: float = 0.20         # 滚转扰动频率
```

**速度选择**：
- **搜索速度**：1.0-1.5 m/s，保守搜索
- **巡航速度**：2.0-2.5 m/s，正常跟踪
- **速度权衡**：更快 → 更多覆盖、更短反应时间

**姿态扰动**（case4）：
- 用于验证坐标变换
- 典型：±5° 滚转、±2° 俯仰
- 频率：0.1-0.3 Hz

### 3.1 case1：直缆跟踪基线

**目的**：验证基础 AC 感知链路 + PCA 中心线拟合 + 磁法埋深反演。作为其他场景的比较锚点。

**配置特点**：
- 直线电缆 `(-320,0)→(360,0)`，`burial_depth_m=1.5`
- 标准噪声 `noise_std_nt=0.05`，无静态安装误差
- 50 Hz 工频，电流 800 A，磁采样率 **500 Hz**
- 声呐 `degraded`（prob_detection=0.75）
- **唯一启用埋深反演**（`BurialInversionConfig.enabled=True`，K=11.4329 nT·m/A_rms）

**实测**：平均航向误差 2.32°、TRACK 占比 86%、模式切换 2 次、埋深反演 MAE 0.056 m、Health 96。

---

### 3.2 case2：折线转弯场景

**目的**：验证遗忘因子与中心线拟合在转弯下的更新能力。

**配置特点**：
- 5 控制点 spline 路由 `(-320,0)→(-40,0)→(120,26)→(260,96)→(420,168)`（持续右转）
- `forgetting_factor=0.68`、`fit_history_size=8`（快速适应转弯）
- 低速 `cruise/search=1.05 m/s`、`min_turning_radius_m=2.5`、`max_yaw_rate=38°/s`
- 声呐较好（prob_detection=0.82、position_noise=0.28 m）

**实测**：平均航向误差 3.17°、TRACK 占比 58%、模式切换 2 次、Health 90。

---

### 3.3 case3：高噪声（手机级）场景

**目的**：验证带通/中值滤波、迟滞防假峰、置信度合理下降。

**配置特点**：
- 手机级噪声 `noise_std_nt=150`、`bias_drift=0.15 nt/s`、`nonorthogonality=1.5°`、静态安装误差 (8,−5,18)°
- 直线路由（继承默认）
- 强抗噪参数：`median_window_samples=9`、`hysteresis_fraction=0.14`、`peak_cooldown_s=1.2`、`min_peak_strength_nt=160`
- 声呐弱（prob_detection=0.55、position_noise=0.9 m）；埋深观测含 dropout

**实测**：平均航向误差 0.53°、TRACK 占比 37%、磁峰 55（噪声毛刺被正确降权）、Health 93。

---

### 3.4 case4：姿态扰动场景

**目的**：验证安装矩阵与机体→NED 姿态补偿。

**配置特点**：
- 大姿态扰动：`roll_amplitude=18°`、`pitch_amplitude=12°`，频率 0.06–0.08 Hz
- 大静态安装误差 (20,−12,35)°
- 直线路由（继承默认）

**实测**：平均航向误差 0.52°、TRACK 占比 57%、模式切换 4 次、Health 94。验证坐标变换链在强姿态下仍稳定。

---

### 3.5 case5：正弦缓弯（低频 demo）场景

**目的**：低频（15 Hz）兼容性 + 缓弯航向跟踪。**注意：本场景路由已从直线升级为正弦缓弯**，是直线拟合曲率表达力的边界测试。

**配置特点**：
- `signal.mode="ac_demo"`、`frequency_hz=15`、电流 520 A、带通半宽 5 Hz
- **正弦路由** `cable_route_mode="sine"`，`sine_amplitudes_m=(6.0, 2.5)`、`sine_wavelengths_m=(150.0, 62.0)`
- `min_cable_curvature_radius_m=25`、`validate_curvature_on_build=True`（最小曲率半径实测 31.1 m，满足硬约束）

**历史与决策**：早期路由 minR 22.4 m，直线滑窗在急弯冻结导致 22.8° 误差；曾尝试"独立磁法曲率估计器"被物理信噪比证伪（见 [`09_实施进度与成果.md`](09_实施进度与成果.md) §4），最终通过**放宽正弦波长**（55→62、140→150）把 minR 提到 31.1 m，误差降至 9.85°。

**实测**：平均航向误差 9.85°（< 15° 硬约束）、TRACK 占比 54%、模式切换 2 次、Health 76。

---

### 3.6 case6：声磁融合（连续强弯）场景

**目的**：声呐间歇 + 连续 S 弯下的声磁协同压力测试。

**配置特点**：
- 5 点 spline 路由 `(-220,-4)→(-120,22)→(0,-18)→(120,28)→(220,-2)`（连续 S 弯，minR≈66 m）
- 中等噪声 `noise_std_nt=0.12`，静态安装误差 (5,−3,10)°
- `fit_acceptance_residual_m=10`（更严拟合接受）

**实测**：平均航向误差 27.58°、TRACK 占比 50%、Health 43。**这是已知未收口项**——直线拟合在连续强弯下表达力不足；与 case5 同源但不通过放宽几何收口（融合压力场景应保持几何挑战）。详见 [`09_实施进度与成果.md`](09_实施进度与成果.md) §7。

---

### 3.7 case7：TRACK 后声呐失效 + 国标式 zig-zag 场景

**目的**：验证“初始定位成功后声呐失效”的工况，以及在 TRACK_ACTIVE 阶段继续执行低幅 zig-zag 是否必要。该场景对应“声呐只做初始种子，后续依赖磁感知 + 航位推算沿缆扫线”的部署假设。

**配置特点**：
- 继承 case1 的直线电缆、50 Hz / 800 A、0.05 nT 噪声与埋深反演标定。
- 声呐初始阶段更可靠（`prob_detection=0.85`、`position_noise=0.25 m`、`heading_noise=3°`），用于快速进入 TRACK。
- `sonar.fail_after_track_active=True`：首次进入 TRACK_ACTIVE 后，下一帧起强制声呐 `FORCED_OFFLINE`。
- `tracking.track_active_zigzag_angle_deg=10.0`：TRACK_ACTIVE 不再纯中心线压线，而是保留 10° 低幅 zig-zag；同时保持磁横偏压线修正。

**实测**：平均航向误差 2.31°、TRACK 占比 86.1%、模式切换 2 次、Health 97.9。首次进入 TRACK 时间约 27.75 s，之后声呐置信度最大值为 0.0，说明失效注入生效。

**对照结论**：同样声呐失效但 TRACK 不 zig-zag 的临时对照，平均航向误差相同（2.31°），但 TRACK 段平均横向偏差为 3.26 m，且无跨缆摆动；case7 的 TRACK 段平均横向偏差为 1.21 m，并出现 2 次跨缆符号变化。结论是：**默认 case 不改为强制 TRACK zig-zag；但保留 `track_active_zigzag_angle_deg` 作为国标/测线任务的显式配置项**。

### 3.9 case_maze_sonar / case_maze_sonar_dropout / case_maze_sparse_sonar / case_maze_no_sonar：光滑迷宫往返长电缆

**目的**：构建更接近现实布放的往返迷宫形电缆压力测试。验收口径不是“跑满时长”，而是 AUV 在无强先验下持续跟踪到电缆终点；到达/越过终点后应停止仿真。

**几何约束（当前稳定基线为 1× prototype，用于快速迭代与回归）**：
- `cable_route_mode="serpentine"`：解析式直线 + 圆弧 U-turn，不使用折线硬拐角。
- `maze_turn_count=4`：4 次折返，5 条来/去长边。
- `maze_straight_length_m=300`：每条纵向来/去长边 300 m。
- `maze_turn_radius_m=30`、`maze_lane_spacing_m=100`：满足最低曲率半径 30 m、来回电缆间距 100 m。
- 当间距大于 `2R` 时，U-turn 由两个 90° 圆弧 + 中间直线过渡构成，避免在宽间距处引入隐含折角。
- `duration_s=2600`，`stop_at_cable_endpoint=True`。

**四种传感器变种**：
- `case_maze_sonar`：声呐可用，`prob_detection=0.92`、`max_range=24 m`、`FOV=160°`。
- `case_maze_sonar_dropout`：声呐仅用于初始锁定，首次进入 TRACK_ACTIVE 后强制 `FORCED_OFFLINE`。
- `case_maze_sparse_sonar`：声呐不强制离线，但命中概率降为 `0.20`，`local_path_max_age_s=180s`，用于低频锚点 + 磁跟踪闭环。
- `case_maze_no_sonar`：声呐关闭，仅依赖磁感知 + 航位推算。

**当前实测**（`/Users/auv_user/miniconda3/bin/python tools/visualize.py --maze`）：
- `case_maze_sonar`：Phase D 可观测区域重捕获开启，`route=99.5%`，`stop_reason=endpoint`，endpoint 距离约 11.4 m。
- `case_maze_sonar_dropout`：`mean_err=3.8°` 但 `route=1.5%`、`stop_reason=duration`，说明局部航向指标好不代表完成 maze；当前闭环仍依赖持续声呐补观测。
- `case_maze_sparse_sonar`：`mean_err=24.2°`、`TRACK=36%`、`route=99.5%`、`stop_reason=endpoint`；说明低频全局锚点足够时，可以降低持续声呐依赖。
- `case_maze_no_sonar`：声呐关闭，仍未完成初始捕获，最终路由进度 0.0%；当前记录为物理/信息可观测性边界，不设强通过目标。

**诊断结论（逐帧实测，按 endpoint 验收口径判断）**：

1. **物理层不是瓶颈**：lane 中心磁场 ~1300 nT、相邻 lane（间距 100–200 m）串扰 < 50 nT，远低于 `min_peak=120 nT`，无信号混叠。
2. **首要失锁发生在直线段而非 U-turn**：`case_maze_sonar` 在 t≈105.9s 进入 TRACK 时，lane1 真值航向为 0°，但 `fused_heading≈-19.5°`；之后 TRACK 段长期冻结在约 -16.9°，车辆从 y≈+4m 单调漂到 y≈-44m，t≈262s 退出 TRACK。此时 route progress 约 158–215m，距离 lane1 末端 600m 和 U-turn 入口仍很远。
3. **根因是线性 PCA 的陈旧观测污染 + 声呐脱窗闭环失败**：早期 SEARCH/ALIGN 阶段的高 SNR 点与少量后续点形成有偏直线，进入 TRACK 后声呐很快超出 24m 量程，缺少新的位置观测修正，fitter 保持旧方向，控制器沿错误航向继续漂离。
4. **最小修正未通过 endpoint 验收**：观测年龄窗把 route progress 从 38.2% 降到 17.0%；放大声呐量程到 60m 降到 19.2%；延迟 TRACK 的 100% progress 是最近点投影假阳性，最终横向距路由约 1229m、`stop_reason=duration`，未触发 endpoint。
5. **当前稳定方案**：有声呐 maze 使用 Phase D `ObservableRegionSelector` 选择可观测区域，并通过 `REACQUIRE_REGION` 任务态短时接管；no-sonar 仍关闭区域控制，避免把缺少纵向观测的场景误判为可通过。

**结论**：1× 有声呐 maze 已形成有效 endpoint 基线；case_maze_no_sonar 仍是可观测性边界。后续调优应优先看 endpoint 和真实距离，不再只看 route progress。

## 6. 配置最佳实践

### 6.1 参数调优顺序

1. **先调信号处理**：确保基础感知正确
2. **再调峰值检测**：平衡灵敏度与稳定性
3. **然后调拟合**：优化响应速度与精度
4. **最后调控制**：优化轨迹性能

### 6.2 参数依赖关系

| 参数 | 依赖参数 | 联动关系 |
|------|----------|----------|
| `frequency_hz` | `bandpass_half_width_hz` | 半宽典型为中心频率的 15% |
| `magnetometer_sample_rate_hz` | `frequency_hz` | 采样率应 ≥ 4× 频率 |
| `min_peak_strength_nt` | `noise_std_nt` | 阈值应 ≥ 3× 噪声 |
| `zigzag_width_gain` | `tracking_strength` | 增益影响宽度自适应范围 |

### 6.3 常见陷阱

1. **采样率不足**：
   - 症状：带通滤波失效
   - 解决：提高采样率或降低频率

2. **阈值过低**：
   - 症状：大量假峰值
   - 解决：提高 `min_peak_strength_nt`

3. **遗忘因子过大**：
   - 症状：转弯响应慢
   - 解决：降低 `forgetting_factor`

4. **Zig-zag 宽度过小**：
   - 症状：频繁丢失
   - 解决：提高 `min_zigzag_width_m`

## 9. 总结

场景配置的核心原则：

1. **渐进式**：从简单场景开始，逐步增加复杂度
2. **可重复**：记录配置参数，确保可重现
3. **有目的**：每个场景验证特定假设
4. **可对比**：保持其他参数不变，只改变目标参数

**配置 = 假设的显式表达**
