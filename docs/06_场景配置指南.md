# 场景配置指南

## 1. 配置系统架构

### 1.1 配置对象层次

```
ScenarioConfig
├── SignalConfig (信号配置)
├── SensorConfig (传感器配置)
│   └── HighFidelityMagnetometerConfig (高保真磁力计)
├── SonarConfig (声纳配置)
├── EnvironmentConfig (环境配置)
├── VehicleConfig (AUV 配置)
├── TrackingConfig (跟踪配置)
└── SignalProcessingConfig (信号处理配置)
```

### 1.2 配置来源

1. **代码默认值**：`config/__init__.py` 中的 dataclass 默认值
2. **场景预设**：`build_default_scenarios()` 中的显式覆盖
3. **命令行参数**：运行时动态修改

## 2. 核心配置类别

### 2.1 信号配置 (SignalConfig)

```python
@dataclass
class SignalConfig:
    mode: str = "ac_50hz"              # dc, ac_50hz, ac_demo, ac_60hz
    frequency_hz: float = 50.0         # AC 频率
    dc_current_a: float = 0.0          # DC 电流幅值
    ac_current_amplitude_a: float = 600.0  # AC 电流幅值
    phase_rad: float = 0.0             # 相位
    bandpass_half_width_hz: float = 8.0 # 带通半宽
```

**模式说明**：
- `dc`：稳恒电流，直接幅值跟踪
- `ac_50hz`：50Hz 工频（中国/欧洲标准）
- `ac_demo`：10-20Hz 实验频率
- `ac_60hz`：60Hz 工频（北美标准）

**关键权衡**：
| 频率 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| DC | 处理简单 | 无载波、易受干扰 | 稳恒电流测试 |
| 10-20Hz | 低采样率可行 | 非标准、环境干扰多 | 实验对照 |
| 50Hz | 标准工频 | 需要较高采样率 | 实际部署 |
| 60Hz | 北美标准 | 需要较高采样率 | 北美应用 |

### 2.2 传感器配置 (SensorConfig)

```python
@dataclass
class SensorConfig:
    magnetometer_sample_rate_hz: float = 200.0  # 采样率
    noise_std_nt: float = 0.05                  # 噪声标准差
    bias_drift_std_nt_per_s: float = 0.01       # 零偏漂移
    nonorthogonality_deg: float = 0.2           # 非正交误差
    static_rotation_euler_deg: Tuple[float,float,float] = (0,0,0)
    dynamic_range_nt: float = 100000.0          # 动态范围
    imu_heading_noise_deg: float = 0.1          # IMU 航向噪声
    imu_tilt_noise_deg: float = 0.05            # IMU 倾斜噪声
    weak_signal_threshold_nt: float = 18.0      # 弱信号阈值
    high_fidelity: HighFidelityMagnetometerConfig = ...
```

**采样率选择**：
- **100 Hz**：最低要求，50Hz 信号勉强可处理
- **200 Hz**：默认值，每周期 4 个样本
- **1000 Hz**：高保真模式，更好的带通性能

**噪声水平**：
- **0.05 nT**：工业级磁力计（标准模式）
- **150 nT**：手机级磁力计（case3）
- **0.03 nT + 粉红噪声**：高保真模拟

**弱信号阈值**：
- 用于判断信号是否足够强
- 典型值：15-20 nT
- 影响置信度和导航源选择

### 2.3 高保真磁力计配置 (HighFidelityMagnetometerConfig)

```python
@dataclass
class HighFidelityMagnetometerConfig:
    enabled: bool = False                      # 是否启用
    sampling_rate_hz: float = 1000.0           # 采样率
    bit_depth: int = 24                        # 比特深度
    full_scale_nt: float = 100000.0            # 量程
    auv_static_interference_body_nt: Tuple[float,float,float] = (35,-20,15)
    white_noise_std_nt: float = 0.03           # 白噪声
    pink_noise_std_nt: float = 0.08            # 粉红噪声
    pink_noise_exponent: float = 1.0           # 粉红噪声指数
    impulse_probability: float = 0.002         # 脉冲概率
    impulse_amplitude_nt: float = 60.0         # 脉冲幅度
    impulse_decay_samples: int = 8             # 脉冲衰减
```

**何时启用**：
- 测试手机级传感器性能（`case_hf_phone`）
- 测试工业级传感器性能（`case_hf_industrial`）
- 验证算法对真实传感器特性的鲁棒性

**关键参数影响**：
| 参数 | 增大影响 | 典型值 |
|------|----------|--------|
| pink_noise_std_nt | 低频漂移更明显 | 0.08 nT |
| impulse_probability | 更多假峰值 | 0.002 (0.2%) |
| impulse_amplitude_nt | 假峰值幅度更大 | 60 nT |

### 2.4 声纳配置 (SonarConfig)

```python
@dataclass
class SonarConfig:
    mode: str = "degraded"                    # off, degraded, reliable_absence
    max_range_m: float = 15.0                 # 最大范围
    horizontal_fov_deg: float = 120.0          # 水平视场角
    prob_detection: float = 0.70               # 探测概率
    position_noise_std_m: float = 0.45         # 位置噪声
    heading_noise_deg: float = 5.0             # 航向噪声
    buried_loss_factor: float = 0.08           # 埋深损失系数
    update_rate_hz: float = 8.0                # 更新频率
```

**模式说明**：
- `off`：声纳不可用
- `degraded`：声纳降级（低探测概率、高噪声）
- `reliable_absence`：可靠地报告"无电缆"

**使用场景**：
- **off**：纯磁力计验证
- `degraded`：默认模式，部分辅助
- `reliable_absence`：部署模式下的排除性搜索

### 2.5 环境配置 (EnvironmentConfig)

```python
@dataclass
class EnvironmentConfig:
    cable_waypoints_xy_m: List[Tuple[float,float]]  # 电缆航路点
    burial_depth_m: float = 1.5                     # 埋深
    cable_route_mode: str = "linear"                 # linear, spline, sine
    field_segment_length_m: float = 1.0              # 场离散化长度
    background_field_ned_nt: Tuple[float,float,float] = (25000,0,42000)
    nominal_route_heading_deg: float = 0.0           # 名义航向
```

**电缆路由模式**：
- `linear`：直线连接航路点
- `spline`：样条平滑（case2 转弯场景）
- `sine`：正弦波叠加（特殊测试）

**埋深设置**：
- 影响：磁场衰减、声纳探测
- 典型值：1-2 m
- 更深 → 磁信号弱、声纳难探测

### 2.6 AUV 配置 (VehicleConfig)

```python
@dataclass
class VehicleConfig:
    search_speed_mps: float = 1.5          # 搜索速度
    cruise_speed_mps: float = 2.0          # 巡航速度
    min_turning_radius_m: float = 5.0       # 最小转弯半径
    max_yaw_rate_deg_s: float = 30.0        # 最大偏航速率
    altitude_above_seabed_m: float = 3.0    # 离底高度
    pitch_amplitude_deg: float = 2.0        # 俯仰扰动幅度
    pitch_frequency_hz: float = 0.15        # 俯仰扰动频率
    roll_amplitude_deg: float = 5.0         # 滚转扰动幅度
    roll_frequency_hz: float = 0.20         # 滚转扰动频率
```

**速度选择**：
- **搜索速度**：1.0-1.5 m/s，保守搜索
- **巡航速度**：2.0-2.5 m/s，正常跟踪
- **速度权衡**：更快 → 更多覆盖、更短反应时间

**姿态扰动**（case4）：
- 用于验证坐标变换
- 典型：±5° 滚转、±2° 俯仰
- 频率：0.1-0.3 Hz

## 3. 预设场景详解

### 3.1 case1：标准直线场景

**目的**：验证基础 AC 感知链路

**配置特点**：
- 标准噪声（0.05 nT）
- 直线电缆（0° 航向）
- 50Hz 工频
- 无姿态扰动

**适用场景**：
- 基础功能验证
- 算法调试
- 性能基准

### 3.2 case2：转弯场景

**目的**：验证转弯响应能力

**配置特点**：
- 折线电缆（0° → 90°）
- 样条平滑路由
- 其余同 case1

**关键参数**：
- 遗忘因子：0.72（快速适应）
- 历史容量：16（不过度限制）

**评估指标**：
- 转弯响应延迟：< 2 次穿缆
- 最终航向误差：< 10°

### 3.3 case3：大噪声场景

**目的**：验证噪声鲁棒性

**配置特点**：
- 手机级噪声（150 nT）
- 其余同 case1

**验证点**：
- 带通滤波有效性
- 中值滤波去毛刺
- 迟滞防止假峰值
- 置信度合理下降

### 3.4 case4：姿态扰动场景

**目的**：验证坐标变换

**配置特点**：
- ±10° 滚转、±5° 俯仰
- 0.15-0.20 Hz 扰动频率
- 其余同 case1

**验证点**：
- 安装矩阵正确性
- 姿态补偿有效性
- NED 系异常场稳定

### 3.5 case5：低频 Demo 场景

**目的**：验证低频兼容性

**配置特点**：
- 10-20Hz 频率
- 其余同 case1

**与 case1 的差异**：
- 带通中心频率不同
- RMS 窗口更长（低频）
- 噪声特性不同

### 3.6 case_hf_phone：高保真手机模式

**目的**：模拟真实手机传感器

**配置特点**：
- 高保真模式启用
- 粉红噪声、脉冲噪声
- AUV 干扰
- 200 Hz 采样率

**验证点**：
- 算法对真实传感器的鲁棒性
- 脉冲噪声抑制
- 低频漂移处理

### 3.7 case_hf_industrial：高保真工业模式

**目的**：模拟工业级传感器

**配置特点**：
- 高保真模式启用
- 1000 Hz 采样率
- 低噪声、高精度

**验证点**：
- 高采样率下的性能
- 精度潜力验证
- 与理论极限对比

```python
def generate_noise_sweep():
    scenarios = {}
    for noise_level in [0.01, 0.05, 0.1, 0.5, 1.0, 5.0, 10.0]:
        name = f"noise_sweep_{noise_level:.2f}"
        base = build_default_scenarios()["case1"]
        base.sensor.noise_std_nt = noise_level
        scenarios[name] = base
    return scenarios
```

## 6. 配置最佳实践

### 6.1 参数调优顺序

1. **先调信号处理**：确保基础感知正确
2. **再调峰值检测**：平衡灵敏度与稳定性
3. **然后调拟合**：优化响应速度与精度
4. **最后调控制**：优化轨迹性能

### 6.2 参数依赖关系

| 参数 | 依赖参数 | 联动关系 |
|------|----------|----------|
| `frequency_hz` | `bandpass_half_width_hz` | 半宽典型为中心频率的 15% |
| `magnetometer_sample_rate_hz` | `frequency_hz` | 采样率应 ≥ 4× 频率 |
| `min_peak_strength_nt` | `noise_std_nt` | 阈值应 ≥ 3× 噪声 |
| `zigzag_width_gain` | `tracking_strength` | 增益影响宽度自适应范围 |

### 6.3 常见陷阱

1. **采样率不足**：
   - 症状：带通滤波失效
   - 解决：提高采样率或降低频率

2. **阈值过低**：
   - 症状：大量假峰值
   - 解决：提高 `min_peak_strength_nt`

3. **遗忘因子过大**：
   - 症状：转弯响应慢
   - 解决：降低 `forgetting_factor`

4. **Zig-zag 宽度过小**：
   - 症状：频繁丢失
   - 解决：提高 `min_zigzag_width_m`

## 7. 配置验证清单

创建新场景后，验证：

- [ ] 信号频率与采样率匹配（采样率 ≥ 4× 频率）
- [ ] 峰值阈值高于噪声（阈值 ≥ 3× 噪声）
- [ ] 带通半宽合理（半宽 ≈ 15% × 中心频率）
- [ ] 拟合容量与遗忘因子匹配
- [ ] Zig-zag 宽度在合理范围（2-9 m）
- [ ] 置信度门限合理（低 ≤ 0.4，高 ≥ 0.6）
- [ ] 超时参数与场景尺度匹配

## 8. 场景选择指南

| 目标 | 推荐场景 | 理由 |
|------|----------|------|
| 快速验证 | case1 | 最简单、最稳定 |
| 转弯测试 | case2 | 唯一转弯场景 |
| 噪声鲁棒性 | case3, case_hf_phone | 大噪声、真实传感器 |
| 坐标验证 | case4 | 姿态扰动 |
| 低频兼容 | case5 | 10-20Hz |
| 高性能验证 | case_hf_industrial | 高采样率、低噪声 |
| 部署测试 | 任意 + --deployment-mode | 无先验测试 |

## 9. 总结

场景配置的核心原则：

1. **渐进式**：从简单场景开始，逐步增加复杂度
2. **可重复**：记录配置参数，确保可重现
3. **有目的**：每个场景验证特定假设
4. **可对比**：保持其他参数不变，只改变目标参数

**配置 = 假设的显式表达**
