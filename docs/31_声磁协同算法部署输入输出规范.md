# 声磁协同算法部署输入输出规范

## 1. 文档目的

本文面向声磁协同电缆跟踪算法的现实部署、外部仿真器接入与电缆运维数据导出，说明部署前需要准备哪些输入、在线算法会产生哪些输出，以及哪些字段只允许用于离线评估而不能进入在线算法。

本规范对应代码中的稳定外层 API：

- `auv_mag_tracking.api.CableMap`
- `auv_mag_tracking.api.NavigationInput`
- `auv_mag_tracking.api.MagneticInput`
- `auv_mag_tracking.api.SonarInput`
- `auv_mag_tracking.api.CableTrackingOutput`
- `auv_mag_tracking.api.AuvMagTrackingPipeline`

其设计目标不是替代现有仿真与可视化系统，而是把核心感知能力整理成 plug-and-play 组件，便于接入更高级的 AUV 管理系统、外部仿真器（例如 HoloOcean）或实船数据链路。

## 2. 部署输入总览

### 2.1 先验电缆地图

先验电缆地图是算法部署的几何基准。最低要求是本地平面坐标系下的二维折线：

| 字段 | 必填 | 含义 |
|---|---:|---|
| `x_m` | 是 | 电缆地图点在局部 NED/工程坐标系中的 x 坐标 |
| `y_m` | 是 | 电缆地图点在局部 NED/工程坐标系中的 y 坐标 |
| `burial_depth_m` | 否 | 该地图点对应的已知或设计埋深 |
| `confidence` | 否 | 地图点可信度 |
| `source` | 否 | 地图来源，如设计图、历史扫测、人工校正 |

CSV 最小示例：

```csv
x_m,y_m,burial_depth_m
0.0,0.0,1.5
20.0,0.0,1.5
40.0,3.0,1.6
```

`CableMap.from_csv()` 要求 `burial_depth_m` 若出现，则每一行都必须提供，避免埋深数组与地图点错位。若暂时没有埋深先验，应完全省略该列。

GeoJSON 支持 `LineString` 与 `MultiLineString`，坐标系必须在工程配置或 metadata 中单独声明。在线算法默认这些坐标已经转换到本地 NED 或等价局部平面坐标，不在 API 内部执行经纬度投影。

### 2.2 航位推算与导航输入

导航输入由 `NavigationInput` 表达：

| 字段 | 必填 | 含义 |
|---|---:|---|
| `time_s` | 是 | 单调递增时间戳 |
| `position_ned_m` | 是 | AUV 在局部 NED 坐标系下的位置，至少包含 x/y |
| `heading_deg` | 是 | AUV 航向角 |
| `pitch_deg` | 否 | 俯仰角 |
| `roll_deg` | 否 | 横滚角 |
| `speed_mps` | 否 | 速度 |
| `position_std_m` | 否 | 位置标准差或导航置信度代理 |
| `heading_std_deg` | 否 | 航向标准差 |
| `source` | 否 | 导航来源，如 `dr_ins`、`dvl_ins`、`simulator` |

部署 CSV 校验工具要求导航日志至少包含：

```csv
time_s,position_x_m,position_y_m,heading_deg
0.0,0.0,2.0,0.0
0.1,0.1,2.0,0.0
```

时间戳必须单调递增或持平，不允许倒退。

### 2.3 磁力计输入

磁力计输入由 `MagneticInput` 表达：

| 字段 | 必填 | 含义 |
|---|---:|---|
| `time_s` | 是 | 采样块时间戳 |
| `sample_block_nt` | 是 | 三轴磁场采样块，单位 nT |
| `sample_rate_hz` | 是 | 采样率 |
| `sensor_frame` | 否 | 传感器坐标系标识 |
| `quality_flags` | 否 | 饱和、掉样、标定状态等质量标志 |

部署 CSV 校验工具要求磁力计日志至少包含：

```csv
time_s,b_x_nt,b_y_nt,b_z_nt
0.0,12.1,-3.2,44.8
0.1,12.0,-3.0,44.6
```

现实部署前必须明确磁力计安装坐标系、硬铁/软铁标定状态、采样率与时间同步方式。若这些信息缺失，算法可以运行，但输出可信度应在上层系统中降低。

### 2.4 声呐输入

声呐输入由 `SonarInput` 表达：

| 字段 | 必填 | 含义 |
|---|---:|---|
| `time_s` | 是 | 声呐观测时间戳 |
| `relative_position_body_m` | 否 | 电缆相对 AUV 机体系的位置 |
| `heading_deg` | 否 | 声呐估计的局部电缆方向 |
| `confidence` | 是 | 声呐观测置信度 |
| `valid` | 是 | 当前观测是否有效 |

部署 CSV 校验工具要求声呐日志至少包含：

```csv
time_s,rel_x_m,rel_y_m,confidence,valid
0.0,8.0,-1.2,0.9,1
0.1,8.1,-1.1,0.8,1
```

声呐可以在锁定后失效；声磁协同算法的关键目标之一，就是在声呐 dropout 后继续利用磁观测、先验地图和主动探查维持可解释的跟踪状态。

## 3. 在线输出

在线输出由 `CableTrackingOutput` 表达：

| 字段 | 含义 |
|---|---|
| `time_s` | 输出时间戳 |
| `estimated_cable_xy_m` | 估计电缆位置 |
| `cross_track_m` | AUV 相对电缆的横偏距离 |
| `route_progress_m` | 沿先验电缆地图的累计进度 |
| `cable_heading_deg` | 当前电缆方向 |
| `burial_depth_m` | 埋深估计或地图埋深 |
| `burial_sigma_m` | 埋深不确定度 |
| `confidence` | 当前输出置信度 |
| `mode` | 输出模式，如 `track`、`map_fallback` |
| `diagnostics` | 诊断字典，包含数据源、地图 frame、磁输入样本数等 |

完整感知链路和 `RunRecord` 还会提供更细的工程诊断字段。与先验地图闭环相关的字段建议上层系统重点记录：

| 字段 | 含义 |
|---|---|
| `map_frame_progress_m` | D4 map-frame tracker 维护的局部连续路线进度 |
| `map_frame_projection_distance_m` | AUV 到当前 map-frame 投影点的距离，用于识别投影不可信 |
| `map_frame_projection_untrusted` | 基于距离/一致性阈值的 D4 不可信标志 |
| `prior_alignment_accepted` | 本帧观测残差是否被在线先验配准状态接受 |
| `prior_alignment_residual_m` | 真实电缆观测点到当前先验投影点的残差 |
| `prior_alignment_step_m` | 本帧实际注入先验配准状态的平移步长 |
| `prior_alignment_translation_x_m/y_m` | 当前累计先验平移修正 |
| `prior_alignment_rotation_deg` | 当前累计先验旋转修正 |

这些字段的部署含义不同：`map_frame_*` 判断"是否还在正确地图段上连续投影"，`prior_alignment_*` 判断"观测是否正在把先验地图拉向真实电缆"。

最小调用示例：

```python
import numpy as np

from auv_mag_tracking.api import (
    AuvMagTrackingPipeline,
    CableMap,
    MagneticInput,
    NavigationInput,
)
from auv_mag_tracking.config import build_default_scenarios

config = build_default_scenarios()["case1"]
cable_map = CableMap(points_xy_m=np.array([[0.0, 0.0], [100.0, 0.0]]))
pipeline = AuvMagTrackingPipeline(config, cable_map)

output = pipeline.step(
    NavigationInput(time_s=1.0, position_ned_m=np.array([10.0, 2.0, -5.0]), heading_deg=0.0),
    MagneticInput(time_s=1.0, sample_block_nt=np.array([[1.0, 2.0, 3.0]]), sample_rate_hz=10.0),
)
```

当前 API facade 是部署契约层，优先稳定输入输出和地图投影行为；更完整的磁感知、在线先验修正和控制逻辑仍由现有仿真/感知模块承载，并可逐步下沉到该 facade 后方。

## 4. 运维导出格式

面向电缆运维人员，算法输出可导出为三类文件：

- `cable_ops_points.csv`：逐点巡检结果。
- `burial_profile.csv`：沿线埋深剖面。
- `diagnostics.json`：点数、埋深点数、平均置信度等摘要。

导出工具：

```bash
python tools/export_cable_ops_report.py --input api_outputs.jsonl --output-dir results/cable_ops_report
```

`api_outputs.jsonl` 可以逐行保存 `CableTrackingOutput` 等价字段，例如：

```json
{"time_s": 1.0, "estimated_cable_xy_m": [10.0, 0.0], "cross_track_m": 2.0, "route_progress_m": 10.0, "cable_heading_deg": 0.0, "burial_depth_m": 1.5, "burial_sigma_m": null, "confidence": 0.8, "mode": "track"}
```

导出的运维字段避免使用仿真真值命名，便于交给非研发人员分析。

## 5. 仿真真值与部署字段边界

现实部署输入不得依赖以下字段：

- `true_nearest_x_m`
- `true_nearest_y_m`
- `true_heading_deg`
- `true_burial_depth_m`
- `evaluation_*` 以外的任何真值派生量

这些字段只允许用于离线评估、论文实验和回归测试。若确实需要把评估真值写入结果文件，必须使用 `evaluation_` 前缀，避免被误接入在线算法。

## 6. 最小部署 Checklist

- 先验电缆地图坐标系已转换到本地 NED 或等价局部平面坐标。
- 导航、磁力计、声呐时间戳已同步，且日志时间单调。
- 磁力计三轴单位为 nT，采样率和安装坐标系已声明。
- 声呐相对位置方向与 AUV 机体系定义一致。
- 埋深列要么每个地图点都有，要么完全省略。
- 上层系统明确区分在线输入与离线评估真值。
- 运行部署输入校验：

```bash
python tools/validate_deployment_inputs.py \
  --cable-map cable_map.csv \
  --navigation navigation.csv \
  --magnetometer magnetometer.csv \
  --sonar sonar.csv
```
