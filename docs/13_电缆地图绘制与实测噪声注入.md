# 电缆地图绘制与实测噪声注入

> **定位**：回答两个落地问题：
> 1. 在假设已有航位推算（DR/INS/DVL/USBL 给出连续 NED 位姿）的前提下，如何紧凑记录识别到的电缆形状；
> 2. 本代码如何注入实测噪声信息，让仿真回归真实平台。
>
> **相关实现**：[`routes/cable_map.py`](../src/auv_mag_tracking/routes/cable_map.py)、[`tools/build_cable_map.py`](../tools/build_cable_map.py)、[`docs/12_实机部署落地指南.md`](12_实机部署落地指南.md)。

---

## 1. 快速确认

### 1.1 是否可以做简易电缆地图绘制？

可以。前提是系统拥有航位推算，能够把每个时刻的 AUV 位置/姿态放到统一的局部 NED 坐标系中。

地图绘制不需要新增复杂 SLAM：本系统已经会输出 `estimated_cable_point_xy_m`（声呐直接定位点、磁峰点或融合中心线局部点）。只要把这些识别出的电缆点按时间落到 NED 平面，再做：

1. 置信度门控；
2. 空间间距降采样；
3. Ramer-Douglas-Peucker（RDP）折线简化；
4. 输出紧凑 polyline JSON；

即可得到一条可复用的“识别电缆地图”。该地图可再转成 `PriorWaypointsRoute`，给后续任务作为稀疏路由先验。

### 1.2 是否支持注入实测噪声信息？

支持。当前代码已有三层噪声注入入口：

| 噪声类型 | 配置入口 | 作用 |
|---|---|---|
| 磁力计白噪声/粉红噪声/脉冲/量化/平台静态干扰 | `HighFidelityMagnetometerConfig` | 模拟真实磁传感器输出 |
| 声呐探测概率/位置噪声/航向噪声/埋深衰减 | `SonarConfig` | 模拟真实声呐定位质量 |
| 埋深通道噪声/掉包 | `SurveyConfig` | 模拟外部埋深观测（可选） |

也就是说，拿到实测日志后，可以标定这些参数，再写入一个“实测平台 case”，用现有 `tools/visualize.py` 和回归流程验证。

## 2. 电缆地图绘制数据流

```
航位推算 pose(t)
    │
    ├── 磁力计/声呐/IMU → perception.update(...)
    │
    ▼
PerceptionState.estimated_cable_point_xy_m
    │
    ▼
RunRecord: estimated_cable_x_m / estimated_cable_y_m / confidence / source
    │
    ▼
CableMapBuilder
    ├── confidence gate
    ├── spacing thinning
    └── RDP simplification
    │
    ▼
cable_map.json  →  CableMap.to_prior_route()
```

核心设计点：

- **航位推算是前提**：没有统一 NED 坐标，识别点只能是局部观测，无法拼成地图。
- **地图只记录识别结果**：默认使用 `estimated_cable_x_m/y_m`，不使用仿真真值 `true_nearest_*`。`--truth-fallback` 只用于合成冒烟测试。
- **紧凑而非稠密**：地图保存的是 polyline waypoints，不保存每帧观测，便于人工检查、版本管理和后续作为路由先验。

## 3. 暴露接口

### 3.1 `CableMapBuilder`

文件：[`routes/cable_map.py`](../src/auv_mag_tracking/routes/cable_map.py)

```python
from auv_mag_tracking.routes import CableMapBuilder

builder = CableMapBuilder(
    min_confidence=0.45,
    min_spacing_m=2.0,
    simplify_tolerance_m=3.0,
    tolerance_band_m=30.0,
)

builder.add_observation(
    position_xy_m=(123.4, -8.0),
    confidence=0.82,
    time_s=42.5,
    source="SONAR",
)

cable_map = builder.build(metadata={"mission": "survey_001"})
cable_map.save_json("results/survey_001/cable_map.json")
prior_route = cable_map.to_prior_route()
```

### 3.2 `build_cable_map_from_record(record, ...)`

```python
from auv_mag_tracking.routes import build_cable_map_from_record
from auv_mag_tracking.viz import simulate_case

record = simulate_case("case5")
cable_map = build_cable_map_from_record(
    record,
    min_confidence=0.45,
    min_spacing_m=2.0,
    simplify_tolerance_m=3.0,
)
cable_map.save_json("results/case5/cable_map.json")
```

### 3.3 `CableMap`

`CableMap` 是最终紧凑地图对象：

```python
from auv_mag_tracking.routes import CableMap

cable_map = CableMap.load_json("results/case5/cable_map.json")
route_prior = cable_map.to_prior_route()
```

输出 JSON 结构：

```json
{
  "schema": "auv_mag_tracking.cable_map.v1",
  "tolerance_band_m": 30.0,
  "waypoints_xy_m": [[...], [...]],
  "metadata": {
    "raw_observation_count": 123,
    "waypoint_count": 8,
    "min_confidence": 0.45,
    "simplify_tolerance_m": 3.0
  }
}
```

## 4. 配套工具

新增 CLI：[`tools/build_cable_map.py`](../tools/build_cable_map.py)

### 4.1 直接从 case 仿真生成地图

```bash
python3 tools/build_cable_map.py --case case5
```

> 若用 `--max-steps` 做快速冒烟，步数需要覆盖到系统稳定识别电缆之后；太早截断会因为没有有效 `estimated_cable_*` 观测而拒绝生成空地图。

默认输出：

```text
results/<时间戳>/case5/
├── record.npz
└── cable_map.json
```

### 4.2 从已有 `record.npz` 生成地图

```bash
python3 tools/build_cable_map.py \
  --record results/<时间戳>/case5/record.npz \
  --out results/<时间戳>/case5/cable_map.json
```

### 4.3 调整紧凑程度

```bash
python3 tools/build_cable_map.py \
  --case case5 \
  --min-confidence 0.50 \
  --min-spacing-m 3.0 \
  --simplify-tolerance-m 4.0 \
  --tolerance-band-m 25.0
```

参数含义：

| 参数 | 含义 | 增大后的影响 |
|---|---|---|
| `--min-confidence` | 观测点置信度门限 | 地图更保守，低质量点更少 |
| `--min-spacing-m` | 相邻保留点最小间距 | 点更少，细节更少 |
| `--simplify-tolerance-m` | RDP 简化容差 | 折线更紧凑，弯道细节可能被压缩 |
| `--tolerance-band-m` | 后续先验路由走廊宽度 | 任务层容忍更宽的路由不确定度 |

## 5. 和本代码如何配合

当前 `viz.RunRecord` 已记录：

- `estimated_cable_x_m`
- `estimated_cable_y_m`
- `confidence`
- `source`

这些通道来自 `PerceptionState.estimated_cable_point_xy_m`，也就是系统认为当前时刻最可信的电缆点。`CableMapBuilder` 只消费这些识别点，不依赖真值。

在实机部署时：

1. `DeploymentSensorSource` 把真实磁/声/IMU/导航打包成 `SensorFrame`；
2. 主循环产生 `PerceptionState.estimated_cable_point_xy_m`；
3. 运行记录器或在线任务代码将该点送入 `CableMapBuilder.add_observation(...)`；
4. 每段任务结束后 `build()` 输出 `cable_map.json`；
5. 下一段任务可通过 `CableMap.to_prior_route()` 将其作为稀疏路由先验。

## 6. 实测噪声注入流程

### 6.1 磁力计噪声

配置入口：`HighFidelityMagnetometerConfig`

建议标定顺序：

1. 电缆断电或远离电缆，采集背景段；
2. 从高频 PSD 平底估计 `white_noise_std_nt`；
3. 从低频 PSD 斜率估计 `pink_noise_std_nt` / `pink_noise_exponent`；
4. 从离群尖峰统计估计 `impulse_probability` / `impulse_amplitude_nt`；
5. 从平台固定偏置估计 `auv_static_interference_body_nt`；
6. 写入一个新 case 的 `sensor.high_fidelity` 配置，并启用 `enabled=True`。

### 6.2 声呐噪声

配置入口：`SonarConfig`

可从实测日志估计：

- `prob_detection`：有效检出比例；
- `position_noise_std_m`：声呐定位点相对人工标注/高精定位的 RMS；
- `heading_noise_deg`：声呐航向估计误差；
- `buried_loss_factor`：埋深导致的检出率衰减。

### 6.3 埋深观测噪声

配置入口：`SurveyConfig`

可标定：

- `burial_depth_noise_std_m`
- `burial_depth_dropout_probability`

没有外部埋深传感器时，该通道保持 `valid=False` 即可，系统仍可使用磁法埋深反演。

## 7. 验证命令

注入实测噪声或生成地图工具后，至少跑：

```bash
python3 -m unittest discover -s tests
python3 tools/visualize.py --all
python3 tools/visualize.py --progress
python3 tools/build_cable_map.py --case case5
```

判定：

- 单测全绿；
- case1–5 指标不回退，case5 仍 < 15°；
- `cable_map.json` 能生成，且 `waypoint_count < raw_observation_count`，说明地图确实完成了紧凑化。
