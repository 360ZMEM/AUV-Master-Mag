### 1.1 三层文档体系（合龙后的写作链路）

```
论文正文层  thesis/paper/        ← 段落式 1–5 章正文（合龙后人工补段落）
   ▲（查证回本树）
工程证据层  thesis/ + 本仓库 docs/  ← 代码符号对照、真实数据、已知偏差
   ▲（追溯原始命令）
原始运行日志  results/ + CSV        ← 单次实验命令、输出、复现入口
```
声磁协同侧的证据层即本仓库 `docs/17–23`，本文（docs/24）及 docs/25–27 是把这些证据**桥接到论文正文层**的中间层。

## 3. 四问题回答

### Q1 — 还需做什么实验？

| 状态 | 实验 | 理由 |
|---|---|---|
| **必做（逻辑严密档）** | DR/INS 边界 sweep 的**多 seed（≥3）复验** | 当前 critical sweep 为 n=1，paper 红线禁止把 n=1 写成统计 |
| **必做（逻辑严密档）** | progress guard **A/B before-after** | 证明 guard 是因果改善而非偶然 |
| **建议（实验充实档）** | 先验几何扭曲 × DR/INS 漂移**二维网格** | 给出失效边界二维等高线，支撑"鲁棒边界"论文图 |
| **建议（实验充实档）** | 全场景图集（`--maze`/`--all`） | 给论文提供 sonar/sparse/dropout 对照的 showcase 图 |

### Q2 — 答辩指标补什么？

- **已产出（路径级健康指标）**：`route_completion_ratio`、`endpoint_completed`、`maze_geometry_passed`、`route_progress_large_jump_count`、`route_progress_max_jump_m`、`track_mean_cross_track_m`、`final_cross_track_m`。这些 [dr_ins_boundary_sweep.py](file:///Users/auv_user/coding/AUV-Master-Mag/tools/dr_ins_boundary_sweep.py) 已直接产出（FIELDNAMES 13 列，定义见 [docs/26 §3](26_声磁协同补充实验与图表执行计划.md)）。
- **需补**：(1) 上述指标的 **mean±std（多 seed）**；(2) progress guard **on/off 对照表**；(3) 失效边界**精确数值 + 触发条件**（[docs/22 §3.7](file:///Users/auv_user/coding/AUV-Master-Mag/docs/22_估计器现状与调参设计文档.md) 已有 sonar_mid 77.6m / heavy 80.0m / sparse_mid 67.4m，需复现确认）。

### Q3 — 什么已足够？

- 单 case 论文图体系（[docs/23](23_论文图清单.md)：20 单 case + 4 showcase + 4 before/after）**已足够**支撑方法图与主结果图。
- route-prior-correction / progress guard / DR/INS / zig-zag 的**机制叙述**（docs/19/20/21/22）**已足够**支撑 paper/03 §3.3、paper/04 §4.3 的深化写作。
- continuous sonar 失效边界（77.6 / 80.0m）**已固化**（docs/22 §3.7），应付毕业档只需复现。

### Q4 — 后续怎么补论文？

- **不改既有 md**；产出 docs/24–27 作为"待整合材料"。
- 人工据本文 §2 映射表把段落补进 paper/03、paper/04，把图引入 paper/05。
- 写作前先读 [docs/25](25_声磁协同与ES-EKF叙事桥接.md) 厘清 ES-EKF 与 route-prior-correction 的上下游关系，避免叙事矛盾。
